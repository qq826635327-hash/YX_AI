## 1. 产品一句话

「1人漫剧」是一款安装在 Windows / macOS / Linux 上的**离线优先** AI 漫剧生产工作台，单人即可完成"剧本 → 分镜 → 角色 / 场景 / 道具资产 → 首尾帧关键图 → 视频"的整套流水线，所有项目数据落到用户本地磁盘的文件夹里。

## 2. 核心特性

| 特性 | 实现方式 |
| --- | --- |
| 离线优先 | 项目即本地文件夹（`project.json + 角色/ + 场景/ + 道具/ + 剧集/分镜/xxx.json + 生成/ + 画布/canvas/*.canvas.json`），不依赖云端 |
| 多模型供应商可插拔 | 出站请求通过 `outbound:request` 走"供应商适配层"，内置支持火山方舟 / 千问 / DeepSeek / OpenAI 兼容 / Claude Agent 等 |
| 角色 / 场景 / 道具 / 分镜 资产化 | 每个资产以独立 JSON 文件持久化，附图（首帧 / 尾帧 / 参考图）存于同目录 `*.png` |
| AI 改剧本 + 抽分镜 | 一次大模型调用把脚本拆成结构化"分镜"列表，落到 `剧集/<集号>/分镜/001.json` |
| AI 生图（图生图、参考图生） | 调图像模型 + 注入 LoRA / 参考图，工作流 JSON 与 ComfyUI 兼容 |
| 画布编辑器 | `画布/*.canvas.json` 是一套自研"图层化卡片"格式，三栏画布、缩放 / 拖拽 / 关联 |
| 任务中心 | 所有 AI 任务（生文 / 生图 / 生视频 / 生音频）走"任务队列 + WebSocket / IPC 广播"实时回传 |
| 插件机制 | 独立 `pluginHost` 子进程 + IPC，第三方插件通过 `ctx.call(...)` 调用宿主能力 |
| 升级与完整性 | app.asar 启动时校验 SHA-256，损坏则提示并阻止运行；可通过 `--ignore-integrity` 跳过 |

## 4. 模块菜单（与左侧导航 1:1 对应）

应用采用**单窗口 + 左侧导航 + 中部工作区 + 右侧属性栏**的三栏布局，左侧固定 7 个一级菜单：

```
首页 / 项目管理   新建项目 / 打开最近 / 切换工作目录 / 退出
项目总览           仪表盘（统计 / 任务 / 资产快链）
剧本 / 分镜       剧集分集 → 剧本正文 → 分镜列表 → 单分镜详情
角色               角色网格 → 角色详情（形象 / 参考图 / 关联分镜）
场景               场景网格 → 场景详情（概念图 / 镜头建议 / 关联分镜）
道具               道具网格 → 道具详情（参考图 / 关键词 / 关联分镜）
画布 / 创作       三栏画布编辑器（从分镜或资产直接进入）
设置              供应商、模型、提示词、ComfyUI、插件、主题、安全
```

另外有**全应用级命令**（顶部命令面板 Ctrl+K 与上下文菜单）：

- 任务中心（随时打开的浮层 / 抽屉）
- 全局搜索（资产 / 分镜 / 角色 / 场景 / 道具）
- 主题切换 / 缩放
- 升级与版本号

## 5. 数据持久化总览

```
<用户文档>/1人漫剧/projects/<项目名>/
├── project.json              # 项目元信息
├── 角色/                     # 角色资产
│   ├── 角色.json             # 全部角色清单
│   ├── 角色/<id>/            # 单角色
│   │   ├── 角色.json         # 角色定义
│   │   ├── 形象.png          # 主形象
│   │   ├── 参考图/           # 参考图集
│   │   └── 生成/             # AI 生成历史
├── 场景/                     # 同上结构
├── 道具/                     # 同上结构
├── 剧集/                     # 剧集
│   ├── 剧集.json             # 全部剧集清单
│   └── 剧集/<集号>/
│       ├── 剧本.md           # 本集剧本正文
│       ├── 剧本.json         # 解析后的结构化剧本
│       └── 分镜/
│           ├── 001.json
│           ├── 001_首帧.png
│           ├── 001_尾帧.png
│           └── ...
├── 画布/                     # 画布
│   └── canvas/<uuid>.canvas.json
├── 生成/                     # 全局生成结果
├── 日志/                     # 项目级日志
└── .wlmj_probe_*             # 写入权限探测临时文件
```

数据全部以**纯 JSON + 邻居目录下的 PNG**形式组织，无数据库，无云端依赖。renderer 通过 `window.electronAPI.fs` + `directoryWatcher` 实时监听并刷新。

## 6. 复刻决策（写代码时必须先定的事）

| 决策点 | 当前产品选择 | 复刻可考虑 |
| --- | --- | --- |
| 桌面壳 | Electron 28+ | 可选 Tauri（更小、需重写主进程为 Rust） |
| 前端框架 | React 18 + Zustand + react-router-dom | 保持或换 Vue 3 + Pinia |
| 构建工具 | Vite 4 | 保持 |
| 持久化 | 纯文件夹 + JSON + 邻居图片 | 可加 SQLite 做索引，但不破坏文件结构 |
| 图像模型 | 出站 HTTP / SSE，支持 火山方舟、OpenAI 兼容、ComfyUI | 同 |
| 视频模型 | 出站 HTTP / SSE（豆包 / 可灵） | 同 |
| 插件进程 | 独立 child_process + IPC，结构化 RpcError 编码 | 保持 |
| 升级 | 启动期 app.asar SHA-256 校验 | 保持，但需配套签名 |
