## 2. 进程模型

Electron 多进程设计，三个角色进程 + 一个插件子进程 + 一个 Claude Agent 子进程：

| 进程 | 入口 | 责任 |
| --- | --- | --- |
| **Main** | `out/main/index.cjs`（95 KB） | 窗口管理、菜单、IPC 总线、文件系统、HTTP 出站、密钥加解密、app 完整性校验、插件宿主调度、Claude Agent 调度 |
| **Preload** | `out/preload/index.cjs`（17 KB） | 唯一被 renderer 可见的桥；通过 `contextBridge.exposeInMainWorld('electronAPI', ...)` 暴露所有 IPC |
| **Renderer** | `out/renderer/index.html` + `out/renderer/assets/index-*.js`（11.7 MB） + `image-host-*.js` | 整个 React 应用；11 MB 是 React 生产版（18.3.1）+ 应用代码的混淆产物 |
| **PluginHost** | `out/main/pluginHost.cjs`（17 KB） | 独立 child_process，第三方插件宿主；通过 IPC 与 Main 通信 |
| **ClaudeAgent** | `out/main/claudeAgent.cjs`（34 KB） | 包装 `@anthropic-ai/claude-agent-sdk`，管理会话、消息、Skill |
| **ImageHost** | 渲染端一个独立 vite bundle | 把图片转 dataURL、生成缩略图、生成 canvas 资源等纯前端图像处理 |

Renderer 与 Main 之间的所有跨进程调用都通过 `electronAPI.<ns>.<method>(...)` 进行，全部走 `ipcRenderer.invoke(...)` / `send(...)` 同步或异步。

## 5. 关键数据流

### 5.1 启动流

```
用户双击 EXE
  └─ Main 进程启动
       ├─ process.resourcesPath/.integrity 校验
       │    └─ 不通过 → 弹错误对话框 + 退出（或 --ignore-integrity 跳过）
       ├─ 启动 PluginHost 子进程
       ├─ 启动 ClaudeAgent 子进程
       ├─ 解析 argv（窗口尺寸、目标路由）
       ├─ BrowserWindow loadFile(renderer/index.html)
       ├─ protocol.handle('app', ...) 注册 app 协议
       └─ 监听 window-all-closed / activate
```

### 5.2 AI 生图流

```
用户在分镜上点"生成首帧"
  → Renderer 端 GenerationService 派任务
  → electronAPI.outbound.request({provider, model, prompt, refImages, params})
  → Main 端 OutboundService
      ├─ 通过 secrets 注入 api key
      ├─ 拼装 HTTP body（OpenAI 兼容 / 火山方舟 / 自定义）
      ├─ 发起 fetch（带超时 + cancel token）
      └─ 流式 chunk 渲染端订阅 outbound:set-chunk-broadcast
  → 渲染端监听 outbound:event 流，把图片 base64 写到 <剧集>/分镜/<id>_首帧.png
  → directoryWatcher 触发并刷新 UI
  → 任务中心追加一条"成功"任务
```

### 5.3 插件调用流

```
插件代码：ctx.call('project.listCharacters', {})
  → PluginHost 进程收到
  → 通过 IPC 转发到 Main
  → Main 找到对应处理函数，调用项目服务
  → 把结果序列化回 PluginHost
  → PluginHost 解析后回给插件代码（Promise resolve）
  失败 → 统一编码 {code, message, data} 返回
```

### 5.4 文件改动实时回传

```
Renderer 写文件 (fs:write-file)
  → 文件落到磁盘
  → Main 端 chokidar / fs.watch 监听（角色/、场景/、道具/、剧集/、分镜/、project.json）
  → 节流 800ms 后批量发送 directory:files-changed
  → Renderer 端 onFilesChanged 回调，局部刷新对应 store
```

## 6. 安全 / 完整性设计

1. **app.asar 完整性**：启动时计算 SHA-256，与 `resources/.integrity` 内的值比对，失败弹错误对话框。
2. **项目目录限制**：不允许在 `ProgramFiles / Windows / 用户安装目录` 内部建项目（避免卸载时被清空）。
3. **API 密钥**：用 `safeStorage`（Windows DPAPI / macOS Keychain / Linux libsecret）加密后落盘到 `userData/.enc`。
4. **renderer 强化**：`contextIsolation: true`、`nodeIntegration: false`、禁用 webview。
5. **media 协议白名单**：`media-protocol:set-allowed-roots` 控制哪些本地目录可被 `app://` 协议代理。
6. **网络白名单**：出站请求只允许 `http:` / `https:` 协议，并经 30x 跳转次数限制（5）。

## 7. 构建 / 发布链（推断）

- 主进程用 esbuild 打包为 cjs 单文件（`out/main/index.cjs` 95 KB 印证）。
- renderer 用 Vite 打包（产物在 `out/renderer/assets/index-*.js`）。
- 通过 `electron-builder` 或 `electron-forge` 打成 NSIS / dmg / AppImage（从 1人漫剧.exe 与文件结构推断）。
- app.asar 内打包 `package.json`，unpacked 仅放必须留盘的原生模块。

## 8. 与第三方服务的关系

| 第三方 | 接入方式 |
| --- | --- |
| 火山方舟（豆包） | `outbound:request` + `volc:*`（凭证 / 资产 / 缩略图） |
| 千问 / DeepSeek / OpenAI 兼容 | `outbound:request`（用户填 baseUrl + key） |
| Claude | `claudeAgent:*`（独立 SDK） |
| 中转图床 | `unifiedUpload`（dataURL → 公共 URL） |
| 阿里云 OSS | `ossUpload`（同上，回退链路） |
| ComfyUI | 工作流导入 + 自有 HTTP 调用（推断走 outbound） |
