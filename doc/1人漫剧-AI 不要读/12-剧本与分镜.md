# 12 剧本与分镜

> 这是产品的核心模块。用户上传剧本 → 解析成结构化剧本 → 一次性"抽分镜" → 在分镜列表里逐条编辑/生成。

## 1. 路由

`/projects/:id/script` → `<ScriptAndShots />`

## 2. 子布局

```
┌─────────────── 顶栏 ────────────────────────────────┐
│ 项目名 │ 任务中心 │ 缩放 │ 主题 │ 设置              │
├──────────┬────────────────────────┬───────────────────┤
│  左侧导航 │  左栏：剧集树          │  右栏：分镜属性   │
│          │  ├── 剧集001 (32)      │  标题             │
│          │  ├── 剧集002 (12)      │  描述             │
│          │  │   ├── 剧本.md       │  角色 / 场景 / 道具│
│          │  │   └── 分镜/         │  镜头参数          │
│          │  └── …                 │  提示词（首/尾/视频）│
│          │  中栏：剧本 / 分镜列表  │  关联资产          │
│          │                       │  生成按钮           │
└──────────┴────────────────────────┴───────────────────┘
```

## 3. 剧集操作

- 新建剧集：弹窗输入"集号 / 名称 / 简介"
- 改名 / 删剧集 / 复制剧集
- 排序（拖拽）

写入 `剧集/剧集.json`，并在 `剧集/<集号>/` 下创建子目录。

## 4. 剧本上传

- 上传 `.md` / `.txt` 文件（`fs:write-file-binary` 解码后存到 `剧本.md`）
- 或者"打开本地剧本"用 `dialog:open-directory` 选文件
- 或者直接编辑（在分屏的左侧 Markdown 编辑器里写）

## 5. 解析剧本

点"解析剧本"→ 一次 `outbound:request` 调用文本模型：

```
prompt = 提示词模板("剧本解析", { rawMarkdown })
→ 返回 { scenes: [...], characters: [...], props: [...] }
→ 写入 剧本.json
```

模板里通过 `{{rawMarkdown}}` 注入全文；模型必须输出结构化 JSON。

## 6. 抽分镜

点"抽分镜"→ 一次 `outbound:request` 文本模型调用：

```
prompt = 提示词模板("分镜拆解", { scriptJson, sceneCount, targetShotCount })
→ 返回 Shots[] （每条含描述、镜头参数、首尾帧 prompt、视频 prompt）
→ 顺序写入 剧集/<集号>/分镜/001.json、002.json…
```

模型可选择"通用 / Grok4 风格 / 批故事版分镜（跨镜接续）"等预设。

## 7. 分镜列表

每行一个分镜卡片：

```
[首帧缩略图] #001 走廊尽头    中景·平视·跟拍  状态 [生成中] [重试]
            描述：…           角色：李修、林远
            镜头参数：…
            提示词：…
```

操作：

- 选中 → 右侧出现属性面板
- 双击标题 → 内联重命名
- 右键菜单：复制 / 删除 / 重新抽分镜（仅此条）/ 打开画布
- 多选 + 拖拽 → 排序
- 批量：选多条 → 批量生成 / 批量配置 / 批量删除

## 8. 单分镜详情

字段（与 JSON 一致）：

- 基础：标题、序号、状态、所属剧集、所属场
- 描述：富文本 / Markdown
- 镜头：cameraSize、cameraAngle、cameraMovement、shotType
- 提示词：
  - `firstFramePrompt`（首帧）
  - `lastFramePrompt`（尾帧）
  - `videoPrompt`（视频）
- 关联：角色 / 场景 / 道具（多选，来自项目内资产）
- 资源：首帧 / 尾帧 / 视频 / 音频预览
- 生成按钮（首帧 / 尾帧 / 视频 / 音频）
- 历史生成（按时间倒序的缩略图列表）

## 9. 关键 IPC

- `outbound:request`（解析剧本 / 抽分镜 / 改写提示词 / 改写镜头建议）
- `outbound:request`（生图 / 生视频 / 生音频）
- `mediaMeta:embed`（生成结果嵌入 wlmjMediaMeta）
- `fs:write-file` × N
- `fs:read-file`（读 剧本.md / 剧本.json / 分镜/NNN.json）
- `directoryWatcher:onFilesChanged`

## 10. 提示词模板

详见 `20-设置-供应商与模型.md`。分镜相关内置模板：

- `剧本解析`：`{{rawMarkdown}}` → 结构化场景
- `分镜拆解`：`{{scriptJson}}` + `{{targetShotCount}}` → 分镜数组
- `首帧生成`：`{{shotDescription}}` + `{{camera}}` + `{{appearancePrompt}}` + `{{styleTag}}`
- `尾帧生成`：类似
- `视频生成`：豆包 / 可灵的 video prompt 风格
- `分镜改写`：把静态分镜改写成"动"风格

## 11. 状态机

```
todo ──生成──> running ──成功──> done
                  │
                  └──失败──> failed ──重试──> running
                  │
                  └──取消──> canceled
```

`status` 字段在 NNN.json 内；`tasks` 独立存于渲染端 store（`useTaskStore`）。

## 12. 性能 / 体验

- 分镜列表虚拟滚动（react-virtual 或自研）
- 缩略图懒加载
- 流式 chunk 在生成时实时刷新右侧预览
- 失败可点"重试"自动沿用上次参数
