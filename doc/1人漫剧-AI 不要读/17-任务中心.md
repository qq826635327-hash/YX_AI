# 17 任务中心

## 1. 入口

- 顶栏"任务中心"按钮（带未读角标）
- 命令面板 Ctrl+K 搜"任务中心"
- 任何生成按钮按下去之后自动弹出（仅第一次）

## 2. 抽屉布局

```
┌──── 任务中心 ─────────────────────────┐
│ [全部] [生文] [生图] [生视频] [失败]   │  ← 顶部 Tab
├────────────────────────────────────────┤
│ ⏳ 分镜 001 首帧   进行中  ·········   │  ← 任务卡
│    进度：2/4 图                       │
│ ✓ 角色 李修 形象    成功   3.2s        │
│ ✗ 场景 001 概念图  失败   [重试]       │
│   错误：upstream 504                  │
└────────────────────────────────────────┘
```

## 3. 数据来源

渲染端 zustand store（`useTaskStore`），不落盘到项目目录，但用 `idb` 在 `userData` 持久化最近 7 天的任务。

```ts
type Task = {
  id: string;
  type: "text" | "image" | "video" | "audio";
  status: "queued" | "running" | "success" | "failed" | "canceled";
  title: string;
  projectPath?: string;
  shotId?: string;
  assetRef?: { kind: "character"|"scene"|"prop"; id: string };
  provider?: string;
  model?: string;
  startedAt: number;
  finishedAt?: number;
  error?: { code: string; message: string };
  chunks?: string[];     // 流式
  result?: {
    path: string;
    publicUrl?: string;
    durationMs: number;
  };
  logs?: string[];
};
```

## 4. 任务生命周期

```
触发点：
- 用户点"生成"
- 批量生成（多次 outbound）
- Claude Agent 工具调用
- 插件异步任务

→ 渲染端 push 一条 task(status=queued)
→ 立即变 running
→ outbound:event 流式 chunk 实时更新 logs
→ 成功 → success，写入 result
→ 失败 → failed，记录 error
→ 取消 → canceled
```

## 5. 批量任务

- 用户选中多条分镜 → "批量生成首帧"
- 渲染端为每条派一个 task，串行 or 并行（用户可配最大并发 1~8）
- 出站请求走 outbound，限流由前端节流 + 后端 service 排队

## 6. 重试 / 取消

- 重试：用同参数重新发一次，复用原 task
- 取消：调 `outbound:cancel(token)`，主进程 abort fetch

## 7. 关键 IPC

- `outbound:request`
- `outbound:cancel`
- `outbound:event`（订阅流式 chunk）
- `claudeAgent:event`（Agent 任务流式）
- `fs:write-file`（成功时落资产）

## 8. 状态展示细节

- 进度条：按"已生成张数 / 总张数"或"已写入字节 / 总字节"计算
- 失败错误码：可点击"复制错误"或"查看完整日志"
- 成功：可直接"打开资产"或"在分镜中查看"
