# 22 设置 · 插件与扩展

## 1. 入口

`/settings/plugins`

## 2. 概念

- 插件是一个独立的 Node.js 包，可被主进程以 child_process 拉起
- 插件通过 `ctx.call(method, params)` 调用宿主能力
- 宿主通过 `pluginHost:event` / `pluginHost:upload-request` / `pluginHost:upload-to-public-url-request` 推事件

## 3. 插件面板

- "已安装插件"列表（来自 `userData/plugins/`）
- "从本地文件夹加载"按钮（`dialog:open-directory`）
- "插件市场"占位（未来可对接）

## 4. 关键 IPC

- `pluginHost:status` / `ensureReady` / `restart` / `call` / `onEvent`
- `pluginHost:onUploadRequest` / `resolveUploadRequest`
- `pluginHost:onUploadToPublicUrlRequest` / `resolveUploadToPublicUrlRequest`
