# 31 插件系统

## 1. 进程模型

```
┌──────────┐    IPC (stdin/stdout line-JSON)   ┌──────────────┐
│ Renderer │  ───────────────────────────────▶  │  PluginHost  │
│          │  ◀───────────────────────────────  │  (child_pro) │
│          │  转发 ctx.call → Main              └──────┬───────┘
│          │                                          │ IPC
│          │                                           ▼
└──────────┘                                    ┌──────────┐
                                                │   Main   │
                                                └──────────┘
```

- `pluginHost.cjs` 启动时扫描 `userData/plugins/*/package.json`，把每个插件 require 起来
- 每个插件在沙箱里拿到一个 `ctx`：

```ts
type PluginContext = {
  call<T = any>(method: string, params?: any): Promise<T>;
  log(...args: any[]): void;
  reportStatus(status: { kind: string; payload: any }): void;
  uploadToOSS(req: { dataUrl: string; category: string }): Promise<{ publicUrl: string }>;
  uploadToPublicUrl(req: { dataUrl: string; category: string; zhongzhuanAuth?: any }): Promise<{ publicUrl: string; provider: string }>;
  // … EventEmitter 风格的事件订阅
};
```

## 2. Host ↔ Main 事件

| 事件 | 方向 | 用途 |
| --- | --- | --- |
| `pluginHost:event` | Main → Renderer | 透传插件 `ctx.reportStatus` |
| `pluginHost:upload-request` | Main → Renderer | 插件调 `ctx.uploadToOSS`，renderer 端监听 + 调 OSS SDK |
| `pluginHost:resolve-upload-request` | Renderer → Main | 调 `pluginHost:resolveUploadRequest` 回传 |
| `pluginHost:upload-to-public-url-request` | Main → Renderer | 插件调 `ctx.uploadToPublicUrl` |
| `pluginHost:resolve-upload-to-public-url-request` | Renderer → Main | 回传 |

## 3. `pluginHost:call`

```ts
window.electronAPI.pluginHost.call(
  method: string,           // 例: "project.listCharacters"
  params: any,
  opts?: { timeoutMs?: number }
)
```

主进程收到后查找方法表（由各 service 注册），调用，回结果。

## 4. 错误

`{ ok: false, error: { code: RpcErrorCodes.X, message, data } }`

## 5. 状态查询

- `pluginHost:status` → `{ ready, pluginCount, lastError?, ... }`
- `pluginHost:ensureReady` / `pluginHost:restart`（用于插件开发热重载）

## 6. 复刻提示

- 建议用 `JSON-RPC over stdio` 而非 socket，更易调试
- `ctx.call` 内部必须捕获异常，绝不抛到主进程 crash
- 上传链路是插件最重的功能，必须有 fingerprint 去重 + 失败回退
