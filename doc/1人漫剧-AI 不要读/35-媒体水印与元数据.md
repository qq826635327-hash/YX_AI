# 35 媒体水印与元数据（wlmjMediaMeta）

## 1. 目的

把"这个图片属于哪个项目哪个分镜、用的什么 prompt / 模型"的信息嵌入到 PNG / MP4 / WAV 字节流末尾，**与文件同生共死**。这样文件被复制到任何地方仍能溯源。

## 2. 容器格式

```
[文件原有字节流]
[Magic = 'WL01' 4B]
[Version = 0x0001 2B]
[Kind = uint8]                          ← 0:asset, 1:oss-cache
[PayloadLen = uint32 BE 4B]
[Payload JSON 字节流]
[CRC32 of Magic..end 4B]
```

## 3. kind 枚举

```ts
type WlmjMetaKind = "asset" | "oss-cache";
const KIND_CODE = { "asset": 0, "oss-cache": 1 };
```

`mediaMeta:read-by-kind` 内部按 kind 过滤并取第一条，节省遍历。

## 4. payload schema

### 4.1 asset

```ts
{
  kind: "asset",
  assetType: "image" | "video" | "audio",
  projectId: string,
  shotId?: string,
  characterId?: string,
  sceneId?: string,
  propId?: string,
  prompt?: string,
  provider?: string,
  model?: string,
  createdAt: string
}
```

### 4.2 oss-cache

```ts
{
  kind: "oss-cache",
  fingerprint: string,        // SHA-1
  provider: "zhongzhuan" | "oss-aliyun",
  publicUrl: string,
  expiresAt: string,
  createdAt: string
}
```

## 5. IPC

```ts
window.electronAPI.mediaMeta.embed({ bytes: ArrayBuffer, meta })
  // → { ok, bytes: ArrayBuffer (新文件字节) }

window.electronAPI.mediaMeta.read({ bytes })
  // → { ok, metas: WlmjMeta[] }

window.electronAPI.mediaMeta.readByKind({ bytes, kind })
  // → { ok, meta: WlmjMeta | null }
```

## 6. 安全

- payload 长度不限，但建议 ≤ 1KB
- 损坏时（CRC 不匹配）`read` 返回 `[]`，**不抛错**
- 写失败（版本不识别）`embed` 返回 `{ok:false, error}`

## 7. 复刻提示

- 嵌入位置在文件末尾不会影响图片 / 视频正常播放
- 抽取时可重写 JS 解析 `wlmjMediaMeta` 的小工具
- 不要把它当成隐私 / 版权机制——任何拿到文件的人都能改
