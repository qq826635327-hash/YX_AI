# PRD-10 · 项目与剧本模块

> **版本**：v1.0 · **Owner**：业务模块负责人 · **关联**：`doc/PRD/01-产品概览与愿景.md`、`doc/开发手册/11-核心数据流.md`

## 1. 模块定位

项目是顶层容器，剧本是整个生产流水线的**数据源头**。本模块负责：
- 项目生命周期（创建 → 活跃 → 归档 → 删除）
- 剧本编辑与版本控制
- AI 解析流水线（剧本 → 实体 / 剧集 / 分镜）

## 2. 业务规则（Business Rules）

### BR-1：项目唯一性
- 同一时刻仅有一个"最近打开"的项目，由前端 `uiStore` 维护
- 项目 `name` 无全局唯一性约束，但磁盘目录通过 `{sanitized_name}_{8位UUID}` 保证唯一

### BR-2：项目状态机
```
[active] ──归档──▶ [archived] ──激活──▶ [active]
   │                                        │
   └─────────── 删除（含文件清理） ────────────┘
```

### BR-3：剧本版本控制
- 每次保存 `raw_text` 自动 `version += 1`
- 解析前保存当前版本，解析失败可回滚

### BR-4：AI 解析三阶段流水线
1. **阶段 1 · 实体提取**：并行调用 LLM 提取 Character / Scene / Prop
   - 同时提取角色（含性别、年龄、外貌提示词）、场景（含镜头建议）、道具
   - 使用数据库中的提示词模板（character/scene/prop 类型）
   - 角色模板包含 `{{style_hint}}` 占位符，注入项目画风
2. **阶段 2 · 章节切分**：将剧本拆为 N 个 Episode
   - 使用提示词模板（episode 类型）
3. **阶段 3 · 分镜拆解**：每集拆为 M 个 Shot，输出 7 段 Prompt
   - 使用提示词模板（shot 类型），输出格式：
     1. 原文对照
     2. 出场人物
     3. 画面
     4. 对话/OS
     5. 首帧提示词（含画风标记）
     6. 尾帧提示词（含画风标记）
     7. 视频提示词（描述首帧到尾帧的视觉运动）
   - 模板包含 `{{style_hint}}`、`{{characters}}`、`{{scenes}}`、`{{props}}` 占位符

### BR-5：画风预设链
- `Project.style_preset` ∈ {default, anime, 3d, ink, realistic, comic}
- 注入 Prompt Template 的 `{{style_hint}}` 变量
- 在 AI 解析阶段 3 自动带入分镜 Prompt
- **画风传递链**：
  ```
  项目 style_preset → 映射为中文画风描述(style_hint)
    → 注入角色模板 {{style_hint}} → LLM 输出含画风的外观描述
    → 注入分镜模板 {{style_hint}} → LLM 输出含画风的首帧/尾帧提示词
  ```
- **画风映射**：anime→二次元动漫风格、3d→3D渲染风格、ink→水墨插画风格、realistic→写实风格、comic→漫画风格

## 3. 功能需求（FR）

### FR-10.1：项目列表页（`/`）

| ID | 需求 | 优先级 |
|---|---|---|
| FR-10.1.1 | 展示所有项目卡片（封面 / 名称 / 描述 / 实体统计 / 最近修改时间） | P0 |
| FR-10.1.2 | 支持关键词搜索与状态过滤（active / archived） | P1 |
| FR-10.1.3 | 新建项目对话框（名称 / 描述） | P0 |
| FR-10.1.4 | 点击卡片进入项目总览；若为最近项目则自动跳转 | P1 |
| FR-10.1.5 | 项目卡片支持归档 / 删除操作（删除需二次确认 + 可选删文件） | P0 |

#### UI 交互细节

- **页面标题**：`AI Drama Studio`，副标题：`选择一个项目开始创作`
- **自动跳转**：如果存在项目，自动跳转到最近操作的项目（按 `updated_at` 倒序）
- **URL 参数**：`?create=1` 强制弹出新建项目对话框
- **空状态**：无项目时显示"还没有项目"引导页
- **项目卡片（ProjectCard）字段**：
  - 项目封面 / 占位图标
  - 项目名称 + 项目描述
  - 统计信息：角色数量、场景数量、道具数量、剧集数量（含分镜总数）
  - 创建时间 / 更新时间
  - "打开项目"按钮
- **新建项目对话框（CreateProjectDialog）**：

  | 字段 | 类型 | 必填 | 说明 |
  |------|------|------|------|
  | 项目名称 | text | ✓ | 唯一 |
  | 项目描述 | textarea | ✗ | 多行 |

  提交后跳转到新项目的总览页。
- **后端 API**：`GET /api/projects`（分页 + 状态过滤 + 关键词搜索）、`POST /api/projects`（自动创建项目根目录）

- **项目头部**：
  - 项目名 + 项目描述 + 状态徽章（active / archived）

  | 场景 | 总数 |
  | 道具 | 总数 |
  | 剧集 | 总数 + 分镜总数 |
  | 任务 | 各状态分布 |

- **快捷入口卡片**：

  | 卡片 | 显示 |
  |------|------|
  | 剧本 | 带解析状态徽章 |
  | 角色 | 带主图就绪状态 |
  | 场景 | 带主图就绪状态 |
  | 道具 | 带主图就绪状态 |
  | 剧集 | 带分镜数量 |

- **最近活动**：最近更新的实体列表（角色 / 场景 / 道具 / 分镜混合）
- **顶栏"项目"按钮**：下拉可快速切换项目

### FR-10.3：剧本页（`/projects/:id/script`）

| ID | 需求 | 优先级 |
|---|---|---|
| FR-10.3.1 | 文本编辑器（60vh 高度，自动保存草稿到本地 storage） | P0 |
| FR-10.3.2 | 顶部画风预设选择器（与 Project.style_preset 双向绑定） | P0 |
| FR-10.3.3 | "保存"按钮调用 `PUT /api/projects/{id}/script` | P0 |
| FR-10.3.4 | "AI 解析"按钮，触发 3 阶段流水线 | P0 |
| FR-10.3.5 | WebSocket 实时显示解析进度（阶段标签 + LLM 流式输出） | P0 |
| FR-10.3.6 | 解析状态徽章（unparsed / parsing / parsed / failed） | P0 |
| FR-10.3.7 | 解析成功后跳转角色页 / 剧集页 | P1 |
| FR-10.3.8 | 解析失败时 LogViewer 红点 + 一键复制错误 JSON | P0 |

#### UI 交互细节

- **顶栏操作**：
  - **保存按钮**：脏检测（dirty），有改动才可点
  - **AI 解析按钮**：触发后端解析流水线
- **画风预设选择器**：位于编辑区顶部，选项为默认 / 二次元动漫 / 3D渲染 / 水墨插画 / 写实 / 漫画，选择后自动保存到项目 `style_preset` 字段
- **状态徽章**：
  - 未解析（灰色）
  - 解析中（黄色脉冲）
  - 已解析（绿色）
  - 解析失败（红色）
- **Tabs 切换**：
  - **编辑 Tab**：大文本编辑框，等宽字体，60vh 高度，字数无上限，修改后自动标记 dirty 状态
  - **解析信息 Tab**：解析时间、解析结果（实体数量统计）、解析错误、JSON 格式原始解析结果（可展开）
- **交互流程**：
  ```
  输入/粘贴文本 → 自动标记 dirty → 点击保存 → 后端 PUT 存储
  → 点击 AI 解析 → 触发后端 pipeline → 状态变为"解析中"
  → 解析完成通过 WebSocket 推送 → 前端自动刷新
  ```
- **后端 API**：`GET /api/projects/{id}/script`、`PUT /api/projects/{id}/script`、`POST /api/projects/{id}/script/parse`
- **WebSocket**：`/ws/script` 推送解析进度

## 4. 用户故事（User Stories）

- **US-10.1**：作为小北，我希望在首页一眼看到所有项目及其规模，以便快速决定继续哪个项目。
- **US-10.2**：作为小北，我希望粘贴完剧本后一键 AI 解析，自动产出所有角色 / 场景 / 道具 / 剧集 / 分镜，避免手工拆解。
- **US-10.3**：作为小北，我希望看到 LLM 实时输出，以判断解析是否偏了，必要时能取消。
- **US-10.4**：作为小北，我希望切换画风预设后，下次解析 / 生成都用新画风，不需要手动改 Prompt。

## 5. 验收标准（Acceptance Criteria，Given-When-Then）

| AC | Given | When | Then |
|---|---|---|---|
| AC-10.1 | 用户在首页 | 点击"新建项目"填完表单提交 | 创建成功并跳转项目总览，磁盘生成 `{name}_{uuid}/` 目录树 |
| AC-10.2 | 项目有未保存的剧本 | 点击"AI 解析" | 系统先自动保存再解析，`parse_status` 从 `none` → `parsing` → `parsed` |
| AC-10.3 | 解析成功 | 进入角色页 | 看到所有提取出的角色，每个角色有 name / description / gender / age |
| AC-10.4 | 解析失败 | 查看 LogViewer | 看到 ERROR 日志 + 结构化 `phase` / `trace_id` |
| AC-10.5 | 用户切换画风预设为 anime | 重新解析剧本 | 分镜 Prompt 中出现 anime 风格的画面描述 |

## 6. 异常场景

| 场景 | 处理 |
|---|---|
| 剧本为空就点解析 | 前端禁用按钮 + Tooltip "剧本不能为空" |
| 解析过程中 LLM API 超时 | 状态变 `failed`，记录 trace_id，保留已产出实体 |
| 项目删除时文件被占用 | 捕获 `PermissionError`，提示用户关闭占用进程 |
| 项目根目录被用户手动移动 | `root_path` 校验失败，提示"项目目录丢失，请重建" |
