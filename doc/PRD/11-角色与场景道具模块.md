# PRD-11 · 角色与场景道具模块

> **版本**：v1.0 · **Owner**：业务模块负责人 · **关联**：`doc/开发手册/04-通用CRUD模式.md`

## 1. 模块定位

角色（Character）、场景（Scene）、道具（Prop）是项目的三类**通用实体**，共享同一套 CRUD 模式与 UI 组件。本模块是 AI 解析流水线的**产物**，也是 AI 生成引擎的**消费者**。

## 2. 业务规则

### BR-1：通用 CRUD 抽象
- 三类实体共用 `business_service.py` 的 `list_by_project / create_entity / update_entity / delete_entity`
- 前端共用 `EntityCard / EntityEditDialog / EntitiesPage`，通过 `entityConfig.ts` 配置字段

### BR-2：实体唯一性
- 同项目内 `name` 唯一（不区分大小写）
- 创建时重名 → 自动追加序号（如 `陆远 (2)`）

### BR-3：实体与磁盘目录
- 每个实体对应一个磁盘目录：`{entity_type_dir}/{entity_name}_{8位UUID}/`
- 目录下含 `images/`、`videos/`、`prompts/` 子目录
- 重命名实体 → 自动重命名目录 + 更新所有 Asset 的 `file_path`

### BR-4：主图关联
- 实体表 `image_asset_id` 指向 Asset 表，表示"展示用主图"
- 设置主图：AssetGallery 点击星标 → `PATCH /entities/{id}` 更新 `image_asset_id`

### BR-5：角色分类
- 角色独有 `char_type` 字段：`protagonist`（主角）/ `supporting`（配角）/ `extra`（群演）
- 角色页支持按 char_type 切换 Tab

### BR-6：场景镜头提示
- 场景独有 `camera_hint` 字段（如"远景 / 室内 / 白天"）
- 注入分镜 Prompt 时作为 `{{scene_camera}}` 变量

### BR-7：级联删除
- 删除实体 → 级联删除其 Asset 记录 + 磁盘文件 + 目录本身
- 使用 `shutil.rmtree(ignore_errors=True)` 包裹 Windows 文件锁

## 3. 功能需求

### FR-11.1：实体列表页（`/projects/:id/{characters|scenes|props}`）

| ID | 需求 | 优先级 |
|---|---|---|
| FR-11.1.1 | 网格卡片布局（3-5 列响应式） | P0 |
| FR-11.1.2 | 角色页显示 `char_type` Tab 过滤（全部 / 主角 / 配角 / 群演） | P0 |
| FR-11.1.3 | 卡片展示：主图 / 名称 / 副标题（角色为性别+年龄，场景为 camera_hint） | P0 |
| FR-11.1.4 | 卡片多选 + 批量操作（批量生成 / 批量删除） | P0 |
| FR-11.1.5 | 卡片悬浮操作：编辑 / 删除 / 生成主图（主图按钮常驻可见，不依赖 hover） | P0 |
| FR-11.1.6 | "新建实体"对话框（name / description / settings / 类型特定字段） | P0 |
| FR-11.1.7 | 空状态引导："点击 AI 解析剧本，自动产出角色 / 场景 / 道具" | P1 |

#### UI 交互细节

- **顶栏操作**：新建实体按钮、多选模式切换（批量操作）、批量生成、批量删除
- **分类 Tabs**（仅角色页）：全部 / 主角（protagonist）/ 配角（supporting）/ 群演（extra）
- **卡片网格**：响应式网格布局（移动端 1 列，平板 2 列，桌面 3-4 列）
- **实体卡片（EntityCard）**：
  - 主图（点击进入详情页）
  - 名称 + 副标题（角色为"性别·年龄"如"男·25岁"；场景为 camera_hint；道具无副标题）
  - 分类徽章（仅角色：主角/配角/群演）
  - 描述（折叠显示）+ 设定（折叠显示）
  - 操作按钮：选中（多选模式）、生成/重新生成主图、编辑、删除（带确认）
- **角色编辑对话框（EntityEditDialog）**：

  | 字段 | 类型 | 必填 | 说明 |
  |------|------|------|------|
  | 名称 | text | ✓ | 项目内唯一 |
  | 性别 | tabs | ✗ | 男 / 女 / 其他（仅角色） |
  | 年龄 | text | ✗ | 如"25岁"、"少年"、"中年"（仅角色） |
  | 分类 | tabs | ✗ | 主角 / 配角 / 群演（仅角色） |
  | 描述 | textarea | ✗ | 3 行 |
  | 设定 | textarea | ✗ | 3 行，用于 AI 生成的提示词 |
  | 镜头建议 | text | ✗ | 如"远景/全景/中景/室内/室外/白天"（仅场景） |

- **后端 API**：
  - `GET /api/projects/{id}/characters?char_type=...`（支持分类过滤）
  - `POST /api/projects/{id}/characters` / `PATCH /{cid}` / `DELETE /{cid}`
  - 场景与道具 API 路径类似，替换 `characters` 为 `scenes` / `props`

### FR-11.2：实体详情页（`/projects/:id/{entities}/:entityId`）

| ID | 需求 | 优先级 |
|---|---|---|
| FR-11.2.1 | 顶部信息区：主图 + 名称 + 描述 + 编辑按钮 | P0 |
| FR-11.2.2 | AssetGallery 网格（所有关联资产，按创建时间倒序） | P0 |
| FR-11.2.3 | 资产操作：设主图 / 删除 / 下载 / 灯箱预览 | P0 |
| FR-11.2.4 | 上传按钮（拖拽 / 点击，限制 200MB，支持 PNG/JPG/WebP/GIF/MP4/WebM） | P0 |
| FR-11.2.5 | "生成"按钮打开 GenerateDialog（target_type = 实体类别，target_id = 实体 ID） | P0 |
| FR-11.2.6 | "同步"按钮调用 `POST /api/assets/sync` 双向修复 DB ↔ 磁盘 | P1 |
| FR-11.2.7 | 关联分镜列表（仅角色 / 场景显示，列出引用了该实体的所有 Shot） | P2 |

#### UI 交互细节

- **页面结构**：
  - **顶部**：返回按钮 + 实体名 + 编辑/删除操作
  - **主图区域**：大图展示 + 生成/重新生成按钮（叠加在图片左上角）+ 点击图片弹出灯箱大图
  - **元信息面板**：分类徽章（仅角色）、描述、设定（用于生成的提示词）
  - **素材画廊（AssetGallery）**：显示该实体关联的所有历史图片/视频
  - **关联分镜**：显示使用了此实体的分镜（角色/场景/道具均有）
- **类型差异**：
  - 角色：顶部图标为用户（Users），有分类徽章
  - 场景：顶部图标为地图（Map），无分类徽章
  - 道具：顶部图标为包裹（Package），无分类徽章

## 4. 用户故事

- **US-11.1**：作为小北，我希望一眼看到所有角色及其主图，以便快速挑选要修改的角色。
- **US-11.2**：作为小北，我希望批量选中多个未生成主图的角色并一键生成，节省重复点击。
- **US-11.3**：作为小北，我希望重命名角色时，磁盘目录和所有资产路径自动同步，避免手动修复。
- **US-11.4**：作为阿远，我希望在详情页看到所有生成图并挑选主图，不满意还能重新生成。

## 5. 验收标准

| AC | Given | When | Then |
|---|---|---|---|
| AC-11.1 | 角色列表有 8 个角色 | 点击"主角"Tab | 仅显示 `char_type = protagonist` 的角色 |
| AC-11.2 | 选中 3 个角色 | 点击"批量生成" | 创建 3 个 GenerationTask，任务中心实时更新 |
| AC-11.3 | 编辑角色名称"陆远"→"陆远 2" | 保存成功 | 磁盘目录 `角色/陆远_xxx` 重命名为 `角色/陆远 2_xxx`，所有 Asset.file_path 更新 |
| AC-11.4 | 删除一个带 12 张资产的角色 | 确认删除 | 角色记录删除 + 12 张资产记录删除 + 磁盘目录删除 |
| AC-11.5 | 上传 500KB PNG 到场景详情页 | 上传完成 | 新 Asset 记录创建，`category = scene`，`target_id = scene_id`，AssetGallery 立即显示 |

## 6. 异常场景

| 场景 | 处理 |
|---|---|
| 创建重名实体 | 后端返回 409，前端提示"名称已存在，已自动改为 {name} (2)" |
| 重命名时目录被占用 | 捕获 `PermissionError`，提示关闭占用进程，保留旧目录名 |
| 上传超 200MB 文件 | 前端拦截，提示"文件大小超限" |
| 删除时磁盘文件被占用 | `ignore_errors=True`，DB 记录仍删除，残留文件留给同步按钮清理 |
