# PRD-12 · 剧集与分镜模块

> **版本**：v1.0 · **Owner**：业务模块负责人 · **关联**：`doc/开发手册/11-核心数据流.md`

## 1. 模块定位

剧集（Episode）是项目的**章节容器**，分镜（Shot）是剧集下的**最小叙事单元**。本模块是剧本解析的产物，也是 AI 生成首帧 / 末帧 / 视频的主要消费者。

## 2. 业务规则

### BR-1：两级层级
```
Project
  └── Episode（剧集，episode_no 1..N）
       └── Shot（分镜，shot_no 01..99）
```

### BR-2：编号规则
- Episode 编号 `episode_no` 由 `sort_order` 派生，1-based
- Shot 编号 `shot_no` 采用 `01/02/03` 格式（2 位补零）
- 创建 / 删除 / 拖拽排序后自动重新编号

### BR-3：引用关系（N:M）
每个 Shot 可引用 0-N 个 Character、0-1 个 Scene、0-N 个 Prop：
- ShotCharacter（多对多中间表）
- ShotScene（多对多中间表，业务上限制 0-1）
- ShotProp（多对多中间表）

### BR-4：7 段式 Prompt 结构
每个 Shot 含 7 个文本段，独立可编辑：
1. **原文**（original_text）：剧本原文片段
2. **角色描述**（characters）：引用角色的外貌汇总
3. **画面描述**（visuals）：AI 生成的画面
4. **台词 / 旁白**（dialogue）
5. **首帧 Prompt**（first_frame_prompt）：用于 AI 生图
6. **末帧 Prompt**（last_frame_prompt）：用于 AI 生图
7. **视频 Prompt**（video_prompt）：用于 AI 生成视频

### BR-5：镜头参数
- `camera_size`：景别（close-up / medium / long / extreme-long）
- `camera_angle`：角度（eye-level / high / low / bird-eye / dutch）
- `camera_movement`：运动（static / pan / tilt / dolly / zoom / crane）

### BR-6：级联删除
- 删除 Episode → 级联删除其所有 Shot → 级联删除 Shot 的所有引用 + 资产

#### UI 交互细节

- **剧集卡片**：
  - 显示：剧集编号、标题、简介、分镜数量
  - 支持折叠/展开分镜列表
  - 新建剧集时编号自动递增（基于项目内最大编号 + 1）
  - 操作：编辑（名称、简介）、删除（带确认，连同所有分镜）
- **分镜卡片**：
  - **标题行**：
    - 分镜号徽章：01、02、03...（两位数格式）
    - 拖拽手柄：左侧 GripVertical 图标
    - 摘要文字
    - 关联引用（内联显示）：角色 Tags（每个带 X 关闭按钮）、场景 Tag、道具 Tags、添加按钮（图标：人物/地图/包裹）
    - 删除按钮（右上角小图标）
  - **插入按钮**：每两个分镜之间有"在此插入"按钮（hover 时显示），列表末尾也有，插入后自动重排所有分镜编号
  - **拖拽排序**：拖拽分镜手柄调整顺序，放下后自动调用 reorder API 持久化，排序后自动重排分镜编号
  - **三列布局（首帧 / 尾帧 / 视频）**：
    - **首帧（first_frame）**：图片占位（生成前显示"生成图片"按钮，生成后显示图片）+ 提示词编辑框 + 状态（none/pending/generating/ready/failed）
    - **尾帧（last_frame）**：与首帧结构一致，用于表示分镜结束时的画面
    - **视频（video）**：视频预览（生成后）+ 提示词编辑框 + 与图片生成类似的状态机
- **实体选择器（EntityPickerDialog）**：
  - 显示当前项目的所有角色/场景/道具
  - 支持搜索过滤
  - 多选模式（角色/道具）/ 单选模式（场景）
- **提示词编辑**：每组素材独立的提示词文本框，修改后自动保存（debounce）
- **生成操作**：首帧/尾帧/视频各自独立的"生成图片"按钮，点击后弹出 GenerateDialog，后端创建 GenerationTask，通过 WebSocket 推送进度，完成后自动刷新该分镜数据
- **后端 API**：
  - 剧集：`GET/POST /api/projects/{id}/episodes`、`GET/PATCH/DELETE /api/projects/{id}/episodes/{eid}`
  - 分镜：`GET/POST /api/episodes/{eid}/shots`、`POST /api/episodes/{eid}/shots/reorder`（拖拽排序）、`GET/PATCH/DELETE /api/shots/{sid}`
  - 引用：`GET /api/shots/{sid}/references`、`POST/DELETE /api/shots/{sid}/characters/{cid}` 等

## 4. 用户故事

- **US-12.1**：作为小北，我希望看到每集的分镜概览，以便快速判断剧情节奏。
- **US-12.2**：作为小北，我希望拖拽分镜调整顺序，自动重编号，不需要手动改 `01/02/03`。
- **US-12.3**：作为小北，我希望在分镜里直接编辑 7 段 Prompt，并能独立触发生成首帧 / 末帧 / 视频。
- **US-12.4**：作为小北，我希望分镜能引用多个角色，并且这些角色的头像在生成时作为参考图注入。

## 5. 验收标准

| AC | Given | When | Then |
|---|---|---|---|
| AC-12.1 | 第 1 集有 10 个分镜 | 拖拽第 3 个分镜到第 7 个位置 | 分镜重编号 01..10，所有关联引用不变 |
| AC-12.2 | 分镜引用 2 个角色 | 点击"生成首帧" | GenerationTask 创建，reference_images 包含 2 个角色主图 |
| AC-12.3 | 编辑首帧 Prompt 保存 | 查看 Shot 详情 | `first_frame_prompt` 字段更新，`first_frame_asset_id` 不变 |
| AC-12.4 | 删除一个 Episode | 确认删除 | 其所有 Shot + ShotCharacter/ShotScene/ShotProp + 资产全部级联清理 |
| AC-12.5 | 删除中间一个 Shot | 触发重编号 | 剩余 Shot 编号连续，无跳号 |

## 6. 异常场景

| 场景 | 处理 |
|---|---|
| 拖拽排序过程中网络失败 | 前端乐观更新 + 失败回滚，提示"排序失败，请重试" |
| 分镜引用了一个已删除的角色 | ShotCharacter 引用失效，前端显示"未知角色"徽章，后端在查询时跳过 |
| 7 段 Prompt 任一为空 | 允许为空，生成时按 Provider 要求填充默认值 |
| 剧集顺序与 `episode_no` 不一致 | 后端强制按 `sort_order` 排序，`episode_no` 仅用于展示 |
