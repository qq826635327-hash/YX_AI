# PRD-15 · 配置中心与设置

> **版本**：v1.0 · **Owner**：业务模块负责人 · **关联**：`doc/开发手册/05-Provider与ComfyUI对接.md`

## 1. 模块定位

配置中心是产品的**专家模式入口**，管理 AI Provider、模型参数、提示词模板、ComfyUI 工作流与插件扩展。

## 2. 业务规则

### BR-1：Provider → Model 两层结构
- 一个 Provider 挂载多个 ProviderModel
- Provider 保存连接信息（base_url / api_key_encrypted / provider_kind）
- ProviderModel 保存模型能力（tags / param_specs / capabilities）

### BR-2：API Key 加密
- 所有 API Key 用 Fernet 对称加密存储（`api_key_encrypted` 字段）
- 加密密钥来源：环境变量 `ADS_ENCRYPTION_KEY`，缺省回退到 `config.yaml` 默认值
- **生产部署必须设置环境变量**，禁止用默认密钥

### BR-3：测试连接
- 调用 Provider 的轻量接口（如 `/models`）验证 API Key 与网络
- 返回 `{ok: bool, latency_ms: int, error?: string}`

### BR-4：提示词模板变量
| 变量 | 来源 | 注入时机 |
|---|---|---|
| `{{script_text}}` | 剧本原文 | AI 解析 |
| `{{style_hint}}` | Project.style_preset | 解析 + 生成 |
| `{{characters}}` | 项目角色列表 | 分镜拆解 |
| `{{scenes}}` | 项目场景列表 | 分镜拆解 |
| `{{props}}` | 项目道具列表 | 分镜拆解 |
| `{{total_shots}}` | 预设分镜数 | 分镜拆解 |

### BR-5：内置模板保护
- `is_builtin = true` 的模板不能删除，只能编辑
- 应用启动时若内置模板缺失，自动 seed

### BR-6：ComfyUI 工作流（规划中）
- `workflow_json` 存储 ComfyUI 节点图
- `input_mapping` 指定 StandardGenerateRequest 字段到节点的映射
- `output_mapping` 指定输出节点到媒体的映射

- **新建/编辑供应商对话框**：

  | 字段 | 类型 | 必填 | 说明 |
  |------|------|------|------|
  | 名称 | text | ✓ | 唯一 |
  | 类型 | select | ✓ | openai / fal / replicate / agnes / 自定义 |
  | API 地址 | text | ✓ | base_url |
  | API Key | password | ✓ | 密文保存，前端展示时脱敏（前 4 位 + 后 4 位 + 中间 8 个星号） |

  - Fernet 对称加密存储在数据库

- **连接测试**：调用 provider 的 health_check 接口，显示延迟、版本、能力验证结果
- **后端 API**：`GET/POST /api/config/providers`、`GET/PATCH/DELETE /api/config/providers/{id}`、`POST /api/config/providers/{id}/test`

## 5. 验收标准

| AC | Given | When | Then |
|---|---|---|---|
| AC-15.1 | 创建 Provider 输入 API Key | 保存 | `api_key_encrypted` 字段为 Fernet 加密串，`GET` 接口不返回 Key |
| AC-15.2 | 测试连接有效 Provider | 点击测试 | 返回 `{ok: true, latency_ms: ~500}` |
| AC-15.3 | 测试连接无效 Key | 点击测试 | 返回 `{ok: false, error: "401 Unauthorized"}` |
| AC-15.4 | 删除内置提示词模板 | 点击删除 | 按钮禁用，Tooltip "内置模板不可删除" |
| AC-15.5 | 设置 Provider A 为 default | 再设置 Provider B 为 default | Provider A 的 `is_default` 自动变为 false |

## 6. 异常场景

| 场景 | 处理 |
|---|---|
| Fernet 加密密钥未配置（用默认） | 启动时 WARNING 日志 + 设置页红条提示"生产环境请设置 ADS_ENCRYPTION_KEY" |
| Provider base_url 格式错误 | 表单校验 + 后端校验，返回 422 |
| 测试连接超时（10s） | 返回 `{ok: false, error: "Timeout"}` |
| 删除 Provider 时仍有 GenerationTask 关联 | 允许删除，旧 Task 的 Provider 信息保留为快照 |
