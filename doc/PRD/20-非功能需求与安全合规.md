## 3. 安全合规（Security）

### 3.1 API Key 加密

| 项 | 规则 |
|---|---|
| 加密算法 | Fernet（AES-128-CBC + HMAC-SHA256） |
| 密钥来源 | 环境变量 `ADS_ENCRYPTION_KEY`，缺省回退 `config.yaml` |
| 字段 | `api_providers.api_key_encrypted` |
| 返回策略 | `GET` 接口永不返回明文，仅返回 `api_key_preview`（前 4 位 + `****`） |
| 生产强制 | 必须设置环境变量，禁止用默认密钥 |

### 3.2 HTTP 认证

| 项 | 规则 |
|---|---|
| BasicAuth | `config.yaml` 可配置 username / password，默认 admin/admin |
| CORS | 仅允许 `http://localhost:5173`（开发）与同源（生产） |
| CSRF | 同源部署，不强制 CSRF token |

### 3.3 输入校验

| 入口 | 校验策略 |
|---|---|
| API 请求体 | FastAPI Pydantic 模型 + `validator` |
| 上传文件名 | `werkzeug.secure_filename` 风格清洗 |
| 上传大小 | 200MB 硬限制 |
| 上传 MIME | 白名单（png/jpg/jpeg/webp/gif/mp4/webm/mov/mkv） |
| SQL 注入 | SQLModel ORM 参数化查询 |

### 3.4 文件操作安全

| 项 | 规则 |
|---|---|
| 路径穿越 | 所有路径通过 `project.root_path` + 相对路径拼接，禁止 `..` |
| 文件锁 | 所有删除操作 try/except PermissionError |
| 级联删除 | 用 `ignore_errors=True` 包裹 |
| SQLite WAL | 写操作使用 `Depends(get_session)`，后台任务用 `session_scope()`，避免长事务 |

### 3.5 第三方依赖安全

| 项 | 规则 |
|---|---|
| 新增依赖 | 必须用户确认，禁止 AI 自动 install |
| 加密库 | 使用 `cryptography`（Fernet） |
| HTTP 客户端 | 使用 `httpx`（默认超时 30s） |

## 8. 运维保障（Operations & Lifecycle）

### 8.1 数据库备份与恢复

| 项 | 规则 |
|---|---|
| 备份方式 | SQLite `.backup` API（WAL-safe 快照，不中断读写） |
| 备份触发 | 应用启动时自动执行一次（startup auto-backup） |
| 手动触发 | `POST /api/config/backup` API 端点 |
| 备份存储 | `data/backups/` 目录，文件名含时间戳 |
| 保留策略 | 可配置保留天数（`config.yaml` → `backup.retention_days`，默认 7 天） |
| 过期清理 | 每次备份后自动扫描并删除超期备份文件 |
| 恢复方式 | 用户手动将备份文件复制到 `data/db/` 并重启 |

### 8.2 数据保留策略

| 数据类型 | 保留期限 | 清理时机 |
|---|---|---|
| `task_logs`（任务日志） | 30 天 | 应用启动时自动清理 |
| `generation_tasks`（已终结任务） | 90 天（status ∈ completed/failed/cancelled） | 应用启动时自动清理 |

### 8.3 WAL 检查点

| 项 | 规则 |
|---|---|
| 触发时机 | 应用启动时（lifespan startup） |
| 检查点模式 | `TRUNCATE`（收缩 WAL 文件至最小） |
| 目的 | 防止 WAL 文件在长时间运行后膨胀，减少磁盘占用和恢复时间 |
| 实现 | `main.py` lifespan startup 中执行 `PRAGMA wal_checkpoint(TRUNCATE)` |

### 8.4 优雅关闭（Graceful Shutdown）

| 项 | 规则 |
|---|---|
| 机制 | `drain_tasks()` 函数，在 server exit 前等待在途任务完成 |
| 超时 | 120 秒（代码硬编码于 `main.py` / `generate.py` 的 `drain_tasks(timeout=120.0)`） |
| 行为 | 停止接受新任务 → 等待在途任务完成或超时 → 关闭数据库连接 → 退出 |
| 超时处理 | 超时后强制退出，未完成的任务状态保留为 `running`（下次启动时由 `reset_task_state` 处理） |

### 8.5 信号量重置（Semaphore Reset）

| 项 | 规则 |
|---|---|
| 触发场景 | `uvicorn --reload` 热重载时，旧进程残留的 `asyncio.Semaphore` 导致新进程信号量计数异常 |
| 机制 | `reset_task_state()` 在 lifespan startup 时清除残留的 Semaphore 状态 |
| 附带行为 | 将上次未完成的 `running` 状态任务重置为 `failed`，附带 `reason: "server restarted"` |

## 9. 已知风险与缓解

| 风险 | 缓解措施 |
|---|---|
| SQLite WAL 大文件写入阻塞 | Phase 4 评估迁移 PostgreSQL；启动时 WAL TRUNCATE 检查点缓解 |
| ~~Huey 进程内模式阻塞 HTTP~~ | **已移除** — Huey 依赖已从 pyproject.toml / config.py / config.yaml 清除，改为用户触发的同步执行 + asyncio |
| 单一 Agnes 供应商锁定 | Provider 抽象层支持多供应商切换 + `fallback_provider` 降级机制 |
| Windows 文件锁 | 所有文件操作 try/except |
| Fernet 默认密钥风险 | 生产强制环境变量 |
| 服务器意外中断导致任务残留 | `drain_tasks()` 优雅关闭 + `reset_task_state()` 启动清理 |
