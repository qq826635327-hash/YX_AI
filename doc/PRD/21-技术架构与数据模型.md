## 3. 数据模型（ER 关系）

```
Project (1) ── (1) ScriptDocument
    │
    ├── (N) Character ── (N) ShotCharacter ── (N) Shot
    │                                          │
    ├── (N) Scene ── (N) ShotScene ────────────┤
    │                                          │
    ├── (N) Prop ── (N) ShotProp ──────────────┤
    │                                          │
    └── (N) Episode ── (N) Shot ───────────────┘
                            │
                            └── (N) Asset (first_frame / last_frame / video)

Character ── (N) Asset (category = character)

ApiProvider (1) ── (N) ProviderModel
Project (1) ── (N) GenerationTask

```

| perf_sessions / perf_events / perf_alerts | 本文件 § 3.2 | 性能监控 |

#### GenerationTask 新增字段（v1.1）

| 字段 | 类型 | 说明 |
|---|---|---|
| `cache_key` | VARCHAR(64) | Prompt 缓存键，用于识别相同 Prompt 的请求以命中缓存 |
| `output_payload` | JSON | 扩展输出载荷，存储 Provider 返回的成本元数据（`cost`、`tokens_used` 等） |
| `auto_retry_count` | INTEGER | 自动重试已执行次数（默认 0） |

#### TasksConfig 新增配置项（v1.2）

| 配置项 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `auto_retry_on_download_fail` | bool | true | 下载失败时是否自动重试 |
| `auto_retry_max_attempts` | int | 3 | 自动重试最大次数 |
| `task_max_age_minutes` | int | 30 | 任务最大年龄（分钟），超时不再自动重试 |

#### 性能监控表（perf_sessions / perf_events / perf_alerts）

| 表 | 字段 | 说明 |
|---|---|---|
| `perf_sessions` | id, session_id(UNIQUE), page, started_at, ended_at, user_agent | 一次页面访问的性能会话 |
| `perf_events` | id, session_id(FK), seq, event_type, name, duration, timestamp, data(JSON) | 单条性能事件（navigation/resource/longtask/paint 等） |
| `perf_alerts` | id, alert_type, threshold_ms, actual_ms, session_id, created_at | 超阈值告警 |

## 4. 后端分层架构

```
app/
├── main.py              # FastAPI 入口 + lifespan（含 WAL checkpoint / 自动备份 / reset_task_state）
├── db.py                # SQLModel engine + session + 优雅关闭辅助
├── core/                # 通用工具（config / encryption / trace / serialization）
├── models/              # SQLModel 数据模型
├── schemas/             # Pydantic 请求/响应模型
├── api/                 # FastAPI routers（薄层，仅参数解析 + 调 service）
├── services/            # 业务逻辑（business_service 是通用 CRUD，backup_service 负责备份/保留/WAL）
├── pipelines/           # AI 解析流水线
│   ├── script_parser.py          # 代理重导出（proxy re-export，保持向后兼容）
│   ├── _orchestrator.py          # 解析主流程编排（原 script_parser 主体）
│   ├── _extraction_stages.py     # 各阶段提取逻辑（角色/场景/道具/分镜）
│   └── _llm_utils.py             # LLM 调用封装（重试/超时/降级）
├── providers/           # AI Provider 适配器（base + handlers + registry）
├── clients/             # 第三方 API 客户端（agnes_client）
└── ws/                  # WebSocket 连接管理
```

> **v1.1 变更说明**：原 Huey 任务队列已移除，任务由 `asyncio.Semaphore` + `create_task` 直接执行；`app/tasks/execute_task.py` 仍保留作为任务执行器。`script_parser.py`（1078 行单体）拆分为 4 个文件，`script_parser.py` 保留为代理重导出入口，所有外部引用无需修改。

### 4.1 关键设计模式

| 模式 | 应用 |
|---|---|
| **通用 CRUD** | `business_service.py` 的 4 个函数覆盖 5 类实体 |
| **translate/parse** | Provider 抽象（子类只实现两个方法） |
| **装饰器注册** | `@register(provider_kind)` 注册 handler |
| **依赖注入** | `Depends(get_session)` 管理 session |
| **上下文变量** | `contextvars` 实现 trace_id 跨层传递 |
| **观察者模式** | WebSocket 推送任务 / 解析 / 日志事件 |
| **代理重导出** | `script_parser.py` 作为 proxy，内部拆分到 `_orchestrator` / `_extraction_stages` / `_llm_utils` |
| **优雅关闭** | `drain_tasks()` + `reset_task_state()` 保证服务重启时任务状态一致 |
| **泛型约束** | `BaseEntity` 泛型 `T extends BaseEntity`，消除 `EntitiesPage` 中的 `any` 类型转换 |

## 6. 通信机制

| 通道 | 协议 | 用途 |
|---|---|---|
| REST | HTTP/JSON | 所有 CRUD + 文件上传下载 |
| WebSocket | /ws/tasks | 任务状态推送 |
| WebSocket | /ws/script | 剧本解析进度推送 |

## 7. 部署架构

```
┌────────────────────────────┐
│  uvicorn (port 8000)       │
│    ├─ FastAPI app          │
│    ├─ 静态文件 (frontend)   │
│    └─ asyncio 任务执行      │
└─────────────┬──────────────┘
              │
              ▼
┌────────────────────────────┐
│  浏览器 (port 5173 dev /    │
│          8000 prod)        │
└────────────────────────────┘
```

- **开发**：前后端分离，Vite dev server (5173) + uvicorn (8000)，Vite 代理 API
- **生产**：前端 build 到 `dist/`，后端 SPA fallback 服务，单进程部署
- **v1.1 变更**：Huey 进程内队列已移除，任务由用户触发 + asyncio 同步执行

## 9. 模块依赖关系（v1.1 新增）

```
backup_service.py
    ├──→ config.py          # 读取 backup.retention_days 配置
    ├──→ db.py              # 获取 engine，执行 .backup API
    └──→ main.py (lifespan) # 启动时自动备份 + 数据保留清理

main.py (lifespan)
    ├──→ backup_service     # startup: 自动备份 + 数据保留清理
    ├──→ generate.py        # startup: reset_task_state()；shutdown: drain_tasks()
    └──→ ws_log_handler      # startup: 注入事件循环

script_parser.py (proxy)
    ├──→ _orchestrator.py   # parse_script_async() 主流程
    ├──→ _extraction_stages.py  # 各阶段提取函数
    └──→ _llm_utils.py      # LLM 调用 + 智能降级

providers/
    ├──→ ProviderHandler._last_result  # Provider 元数据（成本/耗时/token 用量）
    └──→ execute_task.py fallback     # 主 Provider 失败时自动降级到备用
```
