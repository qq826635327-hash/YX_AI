# 01. 全局说明

## 1. 应用启动

应用启动后，浏览器访问 `http://localhost:8000`（或开发模式 `http://localhost:5173`）即可使用。

## 2. 整体布局

```
┌────────────────────────────────────────────┐
│ TopBar（顶栏：项目名、模块切换、用户操作）    │
├──────────┬─────────────────────────────────┤
│          │                                 │
│ Sidebar  │  Content（页面内容）              │
│（侧边栏） │                                 │
│          │                                 │
└──────────┴─────────────────────────────────┘
```

- **TopBar**：固定在顶部，显示当前项目名、模块切换按钮、用户菜单
- **Sidebar**：左侧导航，根据当前模块（项目/任务/配置）动态展示菜单
- **Content**：主内容区，每个页面有统一的 PageContainer 布局

## 3. 全局交互

### 3.1 实时反馈
- **WebSocket 实时推送**：任务进度、剧本解析进度通过 WS 实时反馈
- **轮询降级**：WS 断开时自动启用 5 秒轮询作为降级方案

### 3.2 通知与错误
- **Toast 通知**：操作成功/失败通过右上角 Toast 提示
- **ErrorBoundary**：异常时显示友好降级页，不会白屏
- **API 错误统一处理**：所有 API 错误通过 `ApiError` 类统一抛出

## 4. 主题切换

侧边栏底部的主题按钮支持三态循环：
- **浅色模式**
- **深色模式**
- **跟随系统**

主题选择持久化到 `localStorage`，刷新后保留。

## 5. 数据存储

| 数据类型 | 存储位置 | 加密 |
|----------|---------|------|
| 数据库 | 本地 SQLite（WAL 模式） | - |
| 文件存储 | 项目目录下分类型目录（images/、videos/、prompts/） | - |
| API Key | 数据库 encrypted_api_key 字段 | Fernet 对称加密 |
| 服务端状态 | React Query 缓存 | - |
| 客户端 UI 状态 | Zustand stores | - |
| 主题 | localStorage | - |

## 6. 状态管理

| 状态类型 | 工具 | 例子 |
|----------|------|------|
| 服务端状态 | TanStack Query | 项目列表、角色列表、任务列表 |
| 客户端 UI 状态 | Zustand | 侧边栏折叠、主题、Toast |
| 局部组件状态 | useState/useReducer | 表单输入、对话框开关 |
| 派生状态 | useMemo | 过滤后的列表 |

**规则**：不要把服务端数据存到 Zustand。

## 7. 通用快捷键

| 按键 | 功能 | 状态 |
|------|------|------|
| `ESC` | 关闭对话框/灯箱 | ✅ |
| `Ctrl/Cmd + S` | 保存 | 待实现 |
| `Ctrl/Cmd + K` | 快速搜索 | 待实现 |

## 8. 通用约定

- **路由**：`/api` 前缀
- **响应格式**：成功 `{ data, message? }`，失败 `{ error, message, details? }`
- **时间字段**：所有 datetime 在 API 层统一序列化为字符串
- **删除操作**：所有删除都带确认对话框
- **批量操作**：通过多选模式触发

## 10. API 响应格式规范

> AI 新增或修改 API 时必须遵守以下格式。

### 10.1 成功响应 — 单体

```json
{
  "data": { /* 实体对象 */ },
  "message": "操作成功"
}
```

### 10.2 成功响应 — 列表（无分页）

```json
{
  "data": [ /* 实体数组 */ ],
  "message": "查询成功"
}
```

### 10.3 成功响应 — 列表（分页）

```json
{
  "data": {
    "items": [ /* 实体数组 */ ],
    "total": 100,
    "page": 1,
    "page_size": 20
  },
  "message": "查询成功"
}
```

### 10.4 错误响应

```json
{
  "error": "not_found",
  "message": "项目 xxx 不存在",
  "details": {}
}
```

### 10.5 HTTP 状态码约定

| 状态码 | 含义 | 使用场景 |
|--------|------|---------|
| 200 | 成功 | 所有正常响应 |
| 400 | 请求错误 | 参数格式错误 |
| 403 | 禁止 | 跨项目关联等权限问题 |
| 404 | 未找到 | 资源不存在 |
| 409 | 冲突 | 名称重复等唯一约束违反 |
| 422 | 校验失败 | Pydantic 校验失败 |
| 500 | 服务器错误 | 未捕获异常 |

### 10.6 注意事项

- `data` 字段永远存在（成功时），`message` 可选
- 错误响应中用 `error`（错误类型）+ `message`（人类可读描述）
- 分页响应的 `data` 是对象（含 `items`/`total`/`page`/`page_size`），不是数组
- datetime 字段统一为 ISO 8601 格式（带时区偏移）
- 新增 API 前检查是否有相同功能的现有端点，避免重复
