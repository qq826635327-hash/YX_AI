# 04. 剧本

**路由**：`/projects/{projectId}/script`
**对应文件**：`pages/ScriptPage.tsx`

## 1. 功能描述

剧本模块是整个工作流的数据源。创作者在此粘贴/输入完整剧本，AI 解析后自动抽取角色、场景、道具和剧集结构。

## 2. 页面元素

### 2.1 顶栏操作
- **保存按钮**：脏检测，有改动才可点
- **AI 解析按钮**：触发后端解析流水线

### 2.2 状态徽章
- 未解析（灰色）
- 解析中（黄色脉冲）
- 已解析（绿色）
- 解析失败（红色）

### 2.3 Tabs 切换
- **编辑** Tab：大文本编辑框，等宽字体
- **解析信息** Tab：解析时间、解析结果、解析错误

## 3. 大文本编辑框

- 占满视口 60vh 高度
- 等宽字体，提升剧本可读性
- 字数无上限
- 修改后自动标记 dirty 状态

## 4. 解析结果展示

在解析信息 Tab 中显示：
- 解析时间
- 解析出的实体数量（角色/场景/道具/剧集）
- 错误信息（如有）
- JSON 格式的原始解析结果（可展开）

## 5. 交互流程

```
输入/粘贴文本
  → 自动标记 dirty
  → 点击保存
  → 后端 PUT 存储
  → 点击 AI 解析
  → 触发后端 pipeline
  → 状态变为"解析中"
  → 解析完成通过 WebSocket 推送
  → 前端自动刷新
```

## 6. 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/projects/{id}/script` | 获取剧本 |
| PUT | `/api/projects/{id}/script` | 保存剧本 |
| POST | `/api/projects/{id}/script/parse` | 触发 AI 解析 |

**WebSocket**：`/ws/script` 推送解析进度

## 7. 解析流水线（`pipelines/script_parser.py`）

### 7.1 Phase 1（当前）
基于规则的占位解析（正则表达式抽取）

### 7.2 Phase 2（规划中）
接入 LLM（已预留 LLMConfig），使用结构化输出抽取实体

## 8. 解析输出

解析出的实体自动写入数据库：
- 角色（Character）→ 名称、类型、描述
- 场景（Scene）→ 名称、描述
- 道具（Prop）→ 名称、描述
- 剧集（Episode）→ 编号、标题、简介
- 分镜（Shot）→ 编号、关联角色/场景/道具、提示词

## 9. 数据模型（ScriptDocument）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | str | UUID |
| project_id | str | 所属项目 |
| raw_text | str | 剧本文本 |
| version | int | 版本号（自增） |
| parse_status | str | none / parsing / parsed / failed |
| parse_error | str | 错误信息 |
| parsed_at | datetime | 解析时间 |
| parsed_result | dict | 解析结果 JSON |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |
