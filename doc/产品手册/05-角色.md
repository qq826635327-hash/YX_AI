# 05. 角色

**路由**：
- 列表：`/projects/{projectId}/characters`
- 详情：`/projects/{projectId}/characters/{characterId}`

**对应文件**：
- `pages/CharactersPage.tsx`
- `pages/CharacterDetailPage.tsx`
- 复用组件：[通用组件（EntityCard + EntityEditDialog）](./16-通用组件.md)

## 1. 功能描述

角色模块管理项目中的所有人物角色。每个角色有名片级的元信息和主图（"主图" = 代表该角色的标准图像）。

## 2. 角色列表页

### 2.1 顶栏操作
- 新建角色按钮
- 多选模式切换（批量操作）
- 批量生成（多选后）
- 批量删除（多选后）

### 2.2 分类 Tabs（角色独有）
- 全部
- 主角（protagonist）
- 配角（supporting）
- 群演（extra）

### 2.3 卡片网格
响应式网格布局（移动端 1 列，平板 2 列，桌面 3-4 列）

## 3. 角色卡片（EntityCard）

每张卡片显示：
- 角色主图（点击进入详情页）
- 角色名
- 分类徽章（主角/配角/群演）
- 描述（折叠显示）
- 设定（折叠显示）

**操作按钮**：
- 选中（多选模式）
- 生成/重新生成主图
- 编辑
- 删除（带确认）

## 4. 角色详情页

### 4.1 页面元素
- **顶部**：返回按钮 + 角色名 + 编辑/删除操作
- **主图区域**：
  - 大图展示
  - 生成/重新生成按钮（叠加在图片左上角）
  - 点击图片弹出灯箱大图
- **元信息面板**：
  - 分类徽章
  - 描述
  - 设定（用于生成的提示词）
- **素材画廊**（[AssetGallery](./18-素材画廊.md)）：显示该角色关联的所有历史图片/视频
- **关联分镜**：显示使用了此角色的分镜

## 5. 角色编辑对话框（EntityEditDialog）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 名称 | text | ✅ | 项目内唯一 |
| 分类 | tabs | ❌ | 主角 / 配角 / 群演 |
| 描述 | textarea | ❌ | 3 行 |
| 设定 | textarea | ❌ | 3 行，用于 AI 生成的提示词 |

## 6. 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/projects/{id}/characters?char_type=...` | 列表（支持分类过滤） |
| POST | `/api/projects/{id}/characters` | 新建 |
| GET | `/api/projects/{id}/characters/{cid}` | 详情 |
| PATCH | `/api/projects/{id}/characters/{cid}` | 更新 |
| DELETE | `/api/projects/{id}/characters/{cid}` | 删除（级联清理关联资产和文件） |

## 7. 数据模型（Character）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | str | UUID |
| project_id | str | 所属项目 |
| name | str | 名称 |
| char_type | str | protagonist / supporting / extra |
| description | str | 描述 |
| settings | str | 生成提示词 |
| image_asset_id | str | 主图素材 ID |
| gen_status | str | none / pending / generating / ready / failed |
| sort_order | int | 排序权重 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

**唯一约束**：`(project_id, name)`

## 8. 与场景/道具的差异

| 维度 | 角色 | 场景 | 道具 |
|------|------|------|------|
| 分类 Tabs | ✅ 3 分类 | ❌ | ❌ |
| char_type 字段 | ✅ | ❌ | ❌ |
| 其余功能 | 完全一致 | - | - |

### curl 快速验证

> AI 修改角色模块后，复制以下命令逐一验证。将 `{pid}` 替换为实际项目 ID。

```powershell
# 1. 创建角色
curl -X POST http://127.0.0.1:8000/api/projects/{pid}/characters -H "Content-Type: application/json" -d "{\"name\": \"AI测试角色\", \"char_type\": \"protagonist\", \"description\": \"自动测试\"}"
# 期望: 200, 返回含 id/name/char_type 的 JSON

# 2. 查询列表（验证分类过滤）
curl "http://127.0.0.1:8000/api/projects/{pid}/characters?char_type=protagonist"
# 期望: 200, 返回列表包含刚创建的角色

# 3. 重命名（验证目录同步）
curl -X PATCH http://127.0.0.1:8000/api/projects/{pid}/characters/{cid} -H "Content-Type: application/json" -d "{\"name\": \"改名后角色\"}"
# 期望: 200, 磁盘目录同步重命名

# 4. 删除（验证级联清理）
curl -X DELETE http://127.0.0.1:8000/api/projects/{pid}/characters/{cid}
# 期望: 200, 磁盘目录和 Asset 记录清理

# 5. 同名冲突
curl -X POST http://127.0.0.1:8000/api/projects/{pid}/characters -H "Content-Type: application/json" -d "{\"name\": \"已存在角色\"}"
curl -X POST http://127.0.0.1:8000/api/projects/{pid}/characters -H "Content-Type: application/json" -d "{\"name\": \"已存在角色\"}"
# 第二次期望: 409 或 422
```
