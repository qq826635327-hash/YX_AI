# 08. 剧集结构

**路由**：`/projects/{projectId}/episodes`
**对应文件**：`pages/EpisodesPage.tsx`

## 1. 功能描述

剧集结构是 AI Drama Studio 的核心模块。它将剧本的"分镜级"内容组织为：

```
剧集（Episode）
  └── 分镜（Shot）
       ├── 角色（引用 0-N 个）
       ├── 场景（引用 0-1 个）
       └── 道具（引用 0-N 个）
```

## 2. 剧集管理

### 2.1 剧集列表
- 显示该项目所有剧集
- 卡片显示：剧集编号、标题、简介、分镜数量
- 折叠/展开分镜

### 2.2 新建剧集
- 字段：剧集编号、标题、简介
- 编号自动递增（基于项目内最大编号 + 1）

### 2.3 剧集操作
- 编辑（名称、简介）
- 删除（带确认，连同所有分镜）
- 排序

## 3. 分镜卡片

每条分镜是一张独立卡片，包含：

### 3.1 标题行
- **分镜号徽章**：第 X 集 - Y 分镜
- **摘要文字**
- **关联引用（内联显示）**：
  - 角色 Tags（每个带 X 关闭按钮）
  - 场景 Tag
  - 道具 Tags
  - 添加按钮（图标：人物/地图/包裹）
- **删除按钮**（右上角小图标）

### 3.2 三列布局：首帧 / 尾帧 / 视频

每个分镜有三组素材区：

**首帧（first_frame）**：
- 图片占位（生成前显示"生成图片"按钮，生成后显示图片）
- 提示词编辑框
- 状态：none / pending / generating / ready / failed

**尾帧（last_frame）**：
- 与首帧结构一致
- 用于表示分镜结束时的画面

**视频（video）**：
- 视频预览（生成后）
- 提示词编辑框
- 与图片生成类似的状态机

### 3.3 关联引用面板（已合并到标题行）
- 显示该分镜引用的所有角色/场景/道具
- 点击 X 可移除关联
- 点击 + 按钮打开实体选择器

## 4. 实体选择器（EntityPickerDialog）

- 显示当前项目的所有角色/场景/道具
- 支持搜索过滤
- 多选模式（角色/道具）/ 单选模式（场景）
- 一键添加

## 5. 提示词编辑

- 每组素材独立的提示词文本框
- 修改后自动保存（debounce）
- 提示词内容会作为 AI 生成的输入

## 6. 生成操作

- **首帧/尾帧/视频** 各自独立的"生成图片"按钮
- 点击后弹出 [生成对话框（GenerateDialog）](./19-生成对话框.md)
- 后端创建 `GenerationTask`，通过 WebSocket 推送进度
- 完成后自动刷新该分镜数据

## 7. 后端 API

### 7.1 剧集

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/projects/{id}/episodes` | 列表 |
| POST | `/api/projects/{id}/episodes` | 新建 |
| GET | `/api/projects/{id}/episodes/{eid}` | 详情 |
| PATCH | `/api/projects/{id}/episodes/{eid}` | 更新 |
| DELETE | `/api/projects/{id}/episodes/{eid}` | 删除 |

### 7.2 分镜

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/episodes/{eid}/shots` | 列表 |
| POST | `/api/episodes/{eid}/shots` | 新建 |
| GET | `/api/shots/{sid}` | 详情 |
| PATCH | `/api/shots/{sid}` | 更新 |
| DELETE | `/api/shots/{sid}` | 删除 |

### 7.3 分镜关联引用

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/shots/{sid}/references` | 一次拿全所有引用 |
| POST | `/api/shots/{sid}/characters` | 添加角色关联 |
| DELETE | `/api/shots/{sid}/characters/{cid}` | 移除 |
| POST | `/api/shots/{sid}/scenes` | 添加场景关联 |
| DELETE | `/api/shots/{sid}/scenes/{sid}` | 移除 |
| POST | `/api/shots/{sid}/props` | 添加道具关联 |
| DELETE | `/api/shots/{sid}/props/{pid}` | 移除 |

## 8. 数据模型

### 8.1 Episode
| 字段 | 类型 | 说明 |
|------|------|------|
| id | str | UUID |
| project_id | str | 所属项目 |
| episode_no | int | 剧集编号 |
| title | str | 标题 |
| summary | str | 简介 |
| sort_order | int | 排序权重 |

**唯一约束**：`(project_id, episode_no)`

### 8.2 Shot
| 字段 | 类型 | 说明 |
|------|------|------|
| id | str | UUID |
| project_id | str | 项目 ID |
| episode_id | str | 所属剧集 |
| shot_no | int | 分镜编号 |
| summary | str | 摘要 |
| first_frame_prompt | str | 首帧提示词 |
| first_frame_asset_id | str | 首帧素材 ID |
| first_frame_status | str | 首帧状态 |
| last_frame_prompt | str | 尾帧提示词 |
| last_frame_asset_id | str | 尾帧素材 ID |
| last_frame_status | str | 尾帧状态 |
| video_prompt | str | 视频提示词 |
| video_asset_id | str | 视频素材 ID |
| video_status | str | 视频状态 |
| sort_order | int | 排序权重 |

### curl 快速验证

> 将 `{pid}` 替换为项目 ID，`{eid}` 替换为剧集 ID。

```powershell
# 1. 创建剧集
curl -X POST http://127.0.0.1:8000/api/projects/{pid}/episodes -H "Content-Type: application/json" -d "{\"episode_no\": 1, \"title\": \"第1集\", \"summary\": \"测试\"}"
# 期望: 200

# 2. 创建分镜
curl -X POST http://127.0.0.1:8000/api/episodes/{eid}/shots -H "Content-Type: application/json" -d "{\"shot_no\": 1, \"summary\": \"测试分镜\"}"
# 期望: 200

# 3. 添加角色关联（entity_ids 为数组）
curl -X POST http://127.0.0.1:8000/api/shots/{sid}/characters -H "Content-Type: application/json" -d "{\"entity_ids\": [\"{cid}\"]}"
# 期望: 200

# 4. 跨项目关联（应该失败）
# 用另一个项目的角色 ID 尝试关联
curl -X POST http://127.0.0.1:8000/api/shots/{sid}/characters -H "Content-Type: application/json" -d "{\"entity_ids\": [\"其他项目角色ID\"]}"
# 期望: 403

# 5. 修改 episode_no（验证分镜目录批量重命名）
curl -X PATCH http://127.0.0.1:8000/api/projects/{pid}/episodes/{eid} -H "Content-Type: application/json" -d "{\"episode_no\": 2}"
# 期望: 200, 所有分镜目录重命名

# 6. 删除剧集（验证级联清理）
curl -X DELETE http://127.0.0.1:8000/api/projects/{pid}/episodes/{eid}
# 期望: 200, 分镜目录和关联记录全部清理
```
