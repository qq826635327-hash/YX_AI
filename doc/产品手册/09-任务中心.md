# 09. 任务中心

**路由**：
- 当前项目：`/tasks`
- 全部项目：`/tasks/all`

**对应文件**：
- `pages/TasksPage.tsx`
- `pages/TasksAllPage.tsx`

## 1. 功能描述

任务中心集中展示所有生成任务，便于跟踪进度、定位失败、批量重试。

## 2. 状态过滤

| Tab | 包含状态 |
|-----|----------|
| 全部 | 全部（默认） |
| 进行中 | pending / running / queued |
| 已完成 | succeeded |
| 失败 | failed |
| 已取消 | cancelled |

> 默认展示"全部"状态。

## 3. 任务卡片

每张任务卡片显示：

### 3.1 状态图标
带颜色背景圆圈：
- 排队中（黄）
- 生成中（蓝，带 spinner 动画）
- 已完成（绿）
- 失败（红）
- 已取消（灰）

### 3.2 任务信息
- 目标类型 + 目标 ID 简写
- 状态标签
- 进度百分比（生成中时）
- 错误信息（失败时）

### 3.3 进度条
运行中/排队中显示

### 3.4 操作按钮
| 状态 | 可用操作 |
|------|---------|
| 运行中 | 取消 |
| 失败 | 重试 |
| 成功 | 查看详情 |
| 全部 | 删除 |

## 4. 实时同步

- **WebSocket 推送**：新任务创建/进度更新/状态变化实时收到
- **轮询降级**：5 秒轮询作为兜底
- **状态机驱动**：任务卡片 UI 根据状态自动切换

## 5. 批量操作

- 批量取消（多选运行中任务）
- 批量重试（多选失败任务）

## 6. 当前项目 vs 全部项目

| 入口 | 范围 | URL 参数 |
|------|------|----------|
| 当前项目 | 仅该项目 | `?project_id=xxx` |
| 全部项目 | 跨项目，可按项目过滤 | 无 |

## 7. 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/tasks?project_id=&status=&target_type=&page=&page_size=` | 分页列表 |
| GET | `/api/tasks/{tid}` | 任务详情 |
| POST | `/api/generate/{tid}/cancel` | 取消任务 |
| POST | `/api/generate/{tid}/retry` | 重试任务 |
| POST | `/api/generate/batch` | 批量生成任务 |

## 8. 任务状态机

```
pending → queued → running → succeeded
                    ↓
                  failed ← (可重试 → pending)
                    ↓
                cancelled
```

## 9. 任务字段（GenerationTask）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | str | UUID |
| project_id | str | 所属项目 |
| target_type | str | character / scene / prop / shot_first_frame / shot_last_frame / shot_video |
| target_id | str | 目标实体 ID |
| status | str | pending / queued / running / succeeded / failed / cancelled |
| progress | int | 进度百分比 0-100 |
| prompt | str | 输入提示词 |
| negative_prompt | str | 负向提示词 |
| extra_params | dict | 自定义参数（size, steps, seed 等） |
| provider_id | str | 使用的供应商 |
| workflow_id | str | 使用的工作流 |
| output_asset_id | str | 输出的素材 ID |
| error | str | 错误信息 |
| retry_count | int | 重试次数 |
| started_at | datetime | 开始时间 |
| finished_at | datetime | 完成时间 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

### curl 快速验证

```powershell
# 1. 查询任务列表（分页 + 状态过滤）
curl "http://127.0.0.1:8000/api/tasks?status=running&page=1&page_size=10"
# 期望: 200, 返回 { data: { items: [...], total: N } }

# 2. 提交生成任务（需要 provider_type 和 provider_id）
curl -X POST http://127.0.0.1:8000/api/generate -H "Content-Type: application/json" -d "{\"project_id\": \"{pid}\", \"target_type\": \"character\", \"target_id\": \"{cid}\", \"provider_type\": \"api\", \"provider_id\": \"{provider_id}\", \"prompt\": \"测试生成\"}"
# 期望: 200, 返回 { task_id, status: "running" }

# 3. 取消任务
curl -X POST http://127.0.0.1:8000/api/generate/{tid}/cancel
# 期望: 200, 任务状态变为 cancelled

# 4. 重试失败任务
curl -X POST http://127.0.0.1:8000/api/generate/{tid}/retry -H "Content-Type: application/json" -d "{}"
# 期望: 200, 重新入队

# 5. 重试不存在的任务
curl -X POST http://127.0.0.1:8000/api/generate/nonexistent-id/retry -H "Content-Type: application/json" -d "{}"
# 期望: 404
```
