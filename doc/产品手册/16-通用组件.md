# 16. 通用组件（EntityCard + EntityEditDialog）

**对应文件**：`components/EntityCard.tsx` + `components/EntityEditDialog.tsx`

角色/场景/道具共用同一套卡片和编辑对话框组件，通过 `entityConfig.ts` 注入差异。

---

## 1. 通用资产卡片（EntityCard）

### 1.1 视觉结构

```
┌─────────────────────┐
│   主图占位区域       │  ← 点击进入详情
│  (图标或"生成图片"  │
│   按钮或图片)        │
├─────────────────────┤
│ 名称 / 徽章          │
│ 描述 (折叠)          │
│ 设定 (折叠)          │
│                     │
│ [生成] [编辑] [删除] │
└─────────────────────┘
```

### 1.2 视觉状态

| 状态 | 显示 |
|------|------|
| 已生成 | 显示主图 |
| 未生成 | 显示图标 + "生成" 按钮 |
| 生成中 | 显示 spinner + 进度 |
| 失败 | 显示红色错误图标 |

### 1.3 交互

| 元素 | 行为 |
|------|------|
| 点击主图 | 进入详情页 |
| 生成按钮 | 触发主图生成（仅角色/场景/道具） |
| 编辑按钮 | 弹出 EntityEditDialog |
| 删除按钮 | 带确认对话框 |
| 多选模式 | 显示复选框，触发批量操作 |
| 主图悬停 | 浮起效果（hover transform） |

### 1.4 配置驱动

通过 `entityConfig` 注入：

```ts
{
  name: '角色',
  singular: '角色',
  api: charactersApi,
  icon: Users,
  badgeVariant: 'default',
  detailPath: (id) => `/projects/{pid}/characters/${id}`,
  displayFields: [
    { key: 'description', label: '描述', maxLines: 2 },
    { key: 'settings', label: '设定', maxLines: 2 },
  ],
  getBadge: (entity) => entity.char_type,
}
```

### 1.5 多选模式

- 顶部"多选模式"开关触发
- 卡片左上角显示复选框
- 顶部出现"批量生成"和"批量删除"按钮
- 底部显示"已选 N 项"统计

## 2. 通用编辑对话框（EntityEditDialog）

### 2.1 字段类型

| 类型 | 组件 | 说明 |
|------|------|------|
| text | Input | 单行输入 |
| textarea | Textarea | 多行输入 |
| tabs | Tabs | 标签选择器（用于分类等） |
| select | Select | 下拉选择 |
| number | Input[number] | 数字输入 |

### 2.2 通用字段（所有实体）

| 字段 | 类型 | 必填 |
|------|------|------|
| 名称 | text | ✅ |
| 描述 | textarea | ❌ |
| 设定 | textarea | ❌ |

### 2.3 角色独有字段

| 字段 | 类型 | 必填 |
|------|------|------|
| 分类 | tabs | ❌ |

值：`protagonist` / `supporting` / `extra`

### 2.4 交互

| 按钮 | 行为 |
|------|------|
| 取消 | 关闭对话框 |
| 保存 | 调用对应 API，成功后关闭 |
| ESC | 同取消 |

### 2.5 校验

- 必填字段不能为空
- 名称项目内唯一（后端校验）
- 名称长度限制 1-200
- 描述/设定长度限制 0-2000

### 2.6 模式

- **新建模式**：标题"新建角色"，初始名称为空
- **编辑模式**：标题"编辑角色"，初始值为当前实体

### 2.7 数据回填

- 打开时从 props.entity 读取
- 修改时本地 state 跟踪
- 取消时回滚到原始值
