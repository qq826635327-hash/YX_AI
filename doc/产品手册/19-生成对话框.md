# 19. 生成对话框（GenerateDialog）

**对应文件**：`components/GenerateDialog.tsx`

## 1. 功能描述

统一的多模态素材生成入口。从角色/场景/道具/分镜页触发生成时调用。

## 2. 支持的生成类型

| 触发来源 | target_type |
|----------|-------------|
| 角色主图 | `character` |
| 场景主图 | `scene` |
| 道具主图 | `prop` |
| 分镜首帧 | `shot_first_frame` |
| 分镜尾帧 | `shot_last_frame` |
| 分镜视频 | `shot_video` |

## 3. 页面元素

### 3.1 目标预览
显示当前生成目标的元信息（图片、名称、类型）

### 3.2 参数面板（根据 provider 能力动态渲染）
| 字段 | 类型 | 说明 |
|------|------|------|
| Prompt | textarea | 主提示词 |
| 负向提示词 | textarea | 可选 |
| 尺寸 | select | 1024x1024 等 |
| 参考图 | multi-select | 选自项目内已生成素材 |
| 模型 | select | 选自可用模型 |
| 高级参数 | accordion | 温度、步数、guidance 等 |

### 3.3 底部按钮
- 取消
- 生成（提交后端任务）

## 4. 提交流程

```
点击"生成"
  → 收集参数
  → POST /api/generate
  → 后端创建 GenerationTask 记录
  → 任务入队（huey）
  → worker 异步执行
  → 实时通过 WebSocket 推送进度
  → 完成后创建 Asset 记录
  → 更新目标实体的 image_asset_id
```

## 5. 进度反馈

- 提交后立即跳转到 [任务中心](./09-任务中心.md)
- 通过 WebSocket 推送进度
- 实时显示当前任务状态
- 失败时显示错误信息

## 6. 已知 Phase 1 限制

⚠️ **当前执行是占位实现，不会真生成图**：
- 任务"成功"后目标实体的主图是空的
- 没有任何 prompt 真正发送给外部服务
- 仅用于演示工作流

**Phase 3 接入真实 provider 后**：
- 调用 Agnes / SenseNova / ComfyUI 等
- 真实生成图片/视频
- 进度真实推送

## 7. 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/generate` | 提交单个生成任务 |
| POST | `/api/generate/batch` | 批量提交生成任务 |
| POST | `/api/generate/{task_id}/retry` | 重试失败任务 |
| POST | `/api/generate/{task_id}/cancel` | 取消运行中任务 |

请求体：

```json
{
  "project_id": "uuid",
  "target_type": "shot_first_frame",
  "target_id": "uuid",
  "prompt": "string",
  "negative_prompt": "string",
  "size": "1024x1024",
  "reference_assets": ["uuid"],
  "provider_id": "uuid",
  "workflow_id": "uuid",
  "extra_params": {}
}
```

## 8. 数据流

```
用户点击生成
  ↓
生成对话框
  ↓ POST /api/generate
后端 generation_service
  ↓ 创建 task + 入队
Huey 队列
  ↓ worker 消费
Provider (Phase 3 真实)
  ↓ 进度回调
WS 推送
  ↓
前端 React Query 失效
  ↓
UI 刷新显示新素材
```
