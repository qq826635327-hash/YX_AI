# 00 · 快速开始

> 5 分钟内把项目跑起来。

## 1. 环境要求

| 工具 | 版本 | 说明 |
| --- | --- | --- |
| Python | 3.11+ | 后端 + 任务队列 |
| Node.js | 18+ | 前端构建 |
| OS | Windows 10/11 | macOS / Linux 也能跑（Windows 是主战场） |
| Git | — | 拉代码用 |

## 3. 后端启动

### 方式一：脚本（推荐）

```powershell
cd D:\影序AI\backend\scripts
.\start-dev.bat
```

如果脚本闪退，按下面方式二手动启动。

### 方式二：手动启动

```powershell
cd D:\影序AI\backend

# 1. 激活虚拟环境
.\.venv\Scripts\Activate.ps1

# 2. 直接 uvicorn 启动（不要用 --reload，Windows 下文件监听不可靠）
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

启动成功标志：

```
INFO:     Uvicorn running on http://127.0.0.1:8000
2026-06-20 10:58:07 [root] INFO: === 日志会话开始，日志文件: ..\logs\backend.log ===
2026-06-20 10:58:07 [app.main] INFO: Provider Handler 注册表: {'agnes': 'AgnesHandler', 'openai': 'AgnesHandler'}
```

### 验证后端

| URL | 用途 |
| --- | --- |
| http://127.0.0.1:8000/api/health | 健康检查（JSON） |
| http://127.0.0.1:8000/docs | Swagger UI（自动生成的 API 文档） |
| http://127.0.0.1:8000/api/logs?limit=10 | 历史日志（开发期调试用） |
| ws://127.0.0.1:8000/ws/logs | 实时日志 WebSocket |

## 4. 前端启动

### 方式一：脚本（推荐）

```powershell
cd D:\影序AI\frontend
npm run dev
```

### 方式二：手动

```powershell
cd D:\影序AI\frontend
npm install        # 首次或依赖变化时执行
npm run dev
```

启动成功：

```
  VITE v5.x.x  ready in xxx ms
  ➜  Local:   http://127.0.0.1:5173/
```

### 验证前端

打开 http://127.0.0.1:5173/ ，应该看到首页项目列表。

> **重要**：Vite 代理默认把 `/api` 和 `/ws/` 转发到 `http://127.0.0.1:8000`（`vite.config.ts` 里配的）。
> 局域网访问需要用 `0.0.0.0` host（参考 `08-常见Bug与注意事项.md`）。

## 5. 同时跑后端 + 前端

开两个 PowerShell 窗口，一个跑后端、一个跑前端，**顺序：先开后端**（前端依赖后端 API）。

或者用 `scripts\start-dev.bat`（如果它不闪退）。

## 6. 生产模式

```powershell
# 1. 构建前端
cd D:\影序AI\frontend
npm run build

# 2. 启动后端（会自动托管 frontend/dist/）
cd ..\backend
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

访问 http://localhost:8000 即可。

## 7. 第一个生成流程

1. 打开前端 → 创建一个项目
2. 进入项目 → 剧本 → 粘贴一段剧本 → 点击"解析"
3. 解析完成后自动出现角色/场景/道具/剧集
4. 进入"角色"页 → 点击某角色 → 点击"生成图片" → 选择 Provider → 等 30-60 秒
5. 在"任务中心"看进度；右下角 LogViewer 看实时日志
6. 生成成功后图片自动回填到角色卡片

## 8. 常见问题速查

| 现象 | 排查 |
| --- | --- |
| 后端 8000 端口被占用 | `Get-NetTCPConnection -LocalPort 8000` → `Stop-Process -Id <PID> -Force` |
| 前端 5173 端口被占用 | 改 `vite.config.ts` 的 `server.port` |
| Vite 代理 404 | 确认 `vite.config.ts` 的 `target` 是 8000（不是 8001！） |
| 启动报 `ModuleNotFoundError: fastapi` | 没在 `.venv` 里跑；用 `.\.venv\Scripts\Activate.ps1` |
| 数据库锁死 | 关闭后端；删除 `data/db/app.sqlite-wal` 和 `app.sqlite-shm` |
| Provider API Key 报错 | 去"设置 → API 供应商"检查；Key 不能含未转义的换行 |
| 删除图片报 500 | 看 LogViewer；如果还是失败，跑 `GET /api/logs?level=ERROR` |
