# 07 · 日志监控开发指南

> 这是给 AI 开发助手用的核心功能——可以直接看到前后端错误。实现细节全在这。

## 1. 整体架构

```
Python logging (root logger)
   │  emit(LogRecord)
   ▼
WsLogHandler                  ← backend/app/ws/log_handler.py
   │  asyncio.run_coroutine_threadsafe(broadcast, loop)
   ▼
ConnectionManager             ← backend/app/ws/routes.py
   │  channel = "logs"
   ▼
/ws/logs WebSocket
   │  { type: "log", data: {...}, timestamp: ... }
   ▼
useLogWsSubscription          ← frontend/src/hooks/useLogWsSubscription.ts
   │  pushEntry
   ▼
useLogStore                   ← frontend/src/stores/logStore.ts (最多 500 条)
   │  subscribe
   ▼
LogViewer (浮窗)              ← frontend/src/components/LogViewer.tsx
   │
   └─→ 一键复制为 JSON 给 AI
```

## 2. 后端：WsLogHandler

### 文件：`backend/app/ws/log_handler.py`

```python
class WsLogHandler(logging.Handler):
    BROADCAST_LEVELS = {logging.ERROR, logging.WARNING}

    def __init__(self):
        super().__init__(level=logging.WARNING)
        self._loop = None

    def set_loop(self, loop):
        self._loop = loop

    def emit(self, record):
        if record.levelno not in self.BROADCAST_LEVELS:
            return
        if _ws_manager is None or self._loop is None:
            return
        try:
            data = {
                "level": record.levelname,
                "logger": record.name,
                "message": self.format(record),
                "module": record.module,
                "lineno": record.lineno,
                "func": record.funcName,
            }
            msg = {"type": "log", "data": data, "timestamp": datetime.now(timezone.utc).isoformat()}
            asyncio.run_coroutine_threadsafe(_ws_manager.broadcast("logs", msg), self._loop)
        except Exception:
            pass  # 任何异常都不能让日志广播挂掉

ws_log_handler = WsLogHandler()  # 单例
```

### 关键设计

1. **仅 ERROR/WARNING 广播** —— 避免 INFO 刷屏
2. **`run_coroutine_threadsafe`** —— HTTP 请求处理线程（worker）调 `logger.error()` 时，emit 会被调度到主事件循环
3. **emit 内 `try/except`** —— 日志本身不能因为广播失败而崩
4. **set_loop 必须在 lifespan 中调用** —— 否则跨线程广播会失败

### 接入：main.py

```python
# 在 _setup_logging() 里
ws_log_handler.setFormatter(
    logging.Formatter("%(asctime)s [%(name)s] %(levelname)s: %(message)s", datefmt="%H:%M:%S")
)
logging.getLogger().addHandler(ws_log_handler)

# 在 lifespan 里
loop = asyncio.get_running_loop()
ws_log_handler.set_loop(loop)
set_ws_manager(manager)  # 注入 ConnectionManager 单例
```

## 7. 前端：全局错误捕获

### `main.tsx` 的 `installGlobalErrorHandlers()`

1. **`window.onerror`** —— JS 运行时错误
2. **`unhandledrejection`** —— Promise 拒绝未捕获
3. **`console.error` 拦截** —— 第三方库通过 console.error 报错

```typescript
// 1. JS 错误
window.addEventListener("error", (event) => {
  log.error(event.message, "frontend", `${event.filename}:${event.lineno}`, {
    stack: event.error?.stack,
  });
});

// 2. Promise 拒绝
window.addEventListener("unhandledrejection", (event) => {
  const reason = event.reason;
  const message = reason instanceof Error ? `${reason.name}: ${reason.message}` : String(reason);
  log.error(`Unhandled rejection: ${message}`, "frontend", "promise", {
    stack: reason instanceof Error ? reason.stack : undefined,
  });
});

// 3. console.error 拦截
const originalConsoleError = console.error.bind(console);
console.error = (...args) => {
  // ... 过滤噪音（React DevTools 提示、HMR 提示）
  log.error(text, "frontend", "console.error");
  originalConsoleError(...args);
};
```

## 8. 前端：HTTP 错误自动日志

### `api/client.ts` 的 ky afterResponse 钩子

```typescript
hooks: {
  afterResponse: [
    async (_request, _options, response) => {
      if (!response.ok) {
        const level = response.status >= 500 ? "ERROR" : "WARNING";
        log[level === "ERROR" ? "error" : "warn"](
          `API ${response.status}: ${message}`,
          "api",
          response.url,
          { status: response.status, method: _request.method, url: response.url, body }
        );
        throw new ApiError(message, response.status, body);
      }
    },
  ],
}
```

- **4xx → WARNING**：客户端错误（参数错、权限错、找不到资源）
- **5xx → ERROR**：服务端错误

## 9. 调试流程（给 AI 用的工作流）

1. **打开前端 LogViewer**（`Ctrl+Shift+L` 或右下角按钮）
2. **触发问题操作**（比如点击"生成图片"但失败了）
3. **在 LogViewer 看 ERROR**：
   - `source = api` → 后端返回的错误
   - `source = backend` → 后端 logger 输出
   - `source = frontend` → 浏览器 JS 错误
4. **看 context 字段**：点"详情"展开
5. **点"复制为 JSON"**：把日志贴给 AI，让它诊断
6. **如果需要历史日志**：用 `GET /api/logs?keyword=<关键字>` 查

## 10. 维护注意事项

| 事项 | 原因 |
| --- | --- |
| 修改 `_setup_logging` 的格式 | 会导致 `/api/logs` 解析失败 |
| 修改 WsLogHandler 的 BROADCAST_LEVELS | 影响广播量；如改太多会导致 WS 拥堵 |
| 修改 emit 内部的 try/except | 任何异常都不能让 emit 抛 |
| 改 `set_loop` 逻辑 | 必须保证在 lifespan 中注入；否则跨线程广播会失败 |
| 改 MAX_ENTRIES | 影响前端内存；500 条够用 |
| 修改 console.error 拦截逻辑 | 注意别陷入死循环（log 内部又 console.error） |

## 11. 结构化任务日志（2026-06-21 新增）

### 概述

任务日志从纯文本升级为结构化数据，支持：
- **phase 阶段标记**：system / validate / ref_collect / generate / download
- **event_type 事件类型**：api_request / ref_collect / download / validate / system
- **data_json 结构化数据**：API 请求/响应/耗时/错误等完整信息
- **trace_id 请求追踪**：串联同一任务的多步操作

### TaskLog 模型扩展

```python
class TaskLog(IDMixin, TimestampMixin, table=True):
    task_id: str
    level: str                    # DEBUG / INFO / WARN / ERROR
    message: str
    phase: str | None             # generate / download / ref_collect / validate / system
    event_type: str | None        # api_request / ref_collect / download / validate / system
    data_json: dict | None        # JSON 结构化数据
    trace_id: str | None          # 请求追踪 ID
    data: str | None              # 附加数据（文本，向后兼容）
```

### trace_id 追踪（`core/trace.py`）

```python
from contextvars import ContextVar

_current_trace_id: ContextVar[str] = ContextVar("trace_id", default="")

def new_trace_id() -> str: ...   # 生成新 trace_id 并设置到当前上下文
def get_trace_id() -> str: ...   # 获取当前上下文的 trace_id
def clear_trace_id() -> None: ... # 清除
```

`execute_task_async` 在任务开始时调用 `new_trace_id()`，结束时调用 `clear_trace_id()`。
所有 `write_task_log` / `log_api_call` 等函数自动从上下文获取 trace_id。

### 结构化日志函数

### 脱敏规则

- `api_key` / `apiKey` / `Authorization` → `***REDACTED***`
- `data:image/...` Data URI → `<data:image {N} chars>`（避免日志膨胀）
