> 项目后续要怎么走。当前在哪个阶段、还有什么没做。

## 1. 当前定位

- **架构阶段**：Web 为主，FastAPI + React 18 单体应用
- **核心能力**：项目 → 剧本 → 解析 → 生成 → 资产库

- **持久化**：SQLite（生产应换 PostgreSQL）

## 4. 架构演进建议

| 当前 | 建议演进 | 触发条件 |
| --- | --- | --- |
| FastAPI 单体 | FastAPI + 独立 worker 进程 | 任务耗时变长 |
| SQLite | PostgreSQL | 多人协作 |
| Vite + React SPA | Next.js（SSR + RSC） | SEO / 加载速度 |
| Tailwind 3 | Tailwind 4 | 升级 |
| Zustand | Zustand + Jotai（细粒度） | 状态复杂时 |
| WebSocket | SSE + WebSocket 混合 | 推送数据量大 |
| Huey | Temporal / Celery | 任务流复杂 |

## 5. AI 工作的边界

**AI 应该做**：
- 加新功能（按"我要加新功能去哪改"清单）
- 修 Bug（按 `08-常见Bug与注意事项.md` 排查）
- 重构（在 `business_service` 抽象下加新实体）
- 写测试
- 写文档（必须同步更新本手册和产品手册）

**AI 不应该做**（除非用户明确要求）：
- 改数据库 schema（破坏性变更需要用户确认）
- 改 API 协议（前端要同步改）
- 改 WebSocket 协议（同上）
- 改 Provider 接口（影响所有 Provider 实现）
- 删除 Service / API 路由（要先 grep 看哪里在用）
- 改日志格式（会破坏历史 API 解析）
- 改通用 CRUD 函数签名（影响 5 个实体）

## 6. 文档维护规则

**强制规则**（用户要求）：

> 每次执行任务完成后，**必须**同步更新 `doc/开发手册/CHANGELOG.md` 和相关模块的 `doc/产品手册/` 文档。

具体步骤：

1. **完成任务后**，在 `CHANGELOG.md` 加一条记录（日期、变更、影响范围）
2. **如修改了 API**：更新 `doc/产品手册/A-完整路由表.md`
3. **如修改了前端模块**：更新对应 `doc/产品手册/XX-模块名.md`
4. **如修改了开发相关内容**：更新 `doc/开发手册/XX-主题.md`（如 `02-后端架构.md`、`04-通用CRUD模式.md`）
5. **如发现新 Bug 或踩坑**：更新 `doc/开发手册/08-常见Bug与注意事项.md`
6. **如新增 TODO**：更新本章"紧急 TODO"段落
