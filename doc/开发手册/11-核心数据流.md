## Chain 1：实体 CRUD（角色创建/编辑/删除）

### 总览

```
用户点击保存/删除
  → EntityEditDialog 回调
    → useBusiness hook (useMutation)
      → ky HTTP 请求 (POST/PATCH/DELETE)
        → FastAPI route (api/characters.py)
          → business_service.create/update/delete_entity
            → SQLModel session.commit()
              → 磁盘目录操作 (mkdir / rename / rmtree)
                → invalidateQueries
                  → useQuery 重新拉取
                    → EntityCard 重新渲染
```

```python
def create_entity(session, model, project_id, data):
```

#### 步骤 6：更新（PATCH）特殊处理 — update_entity

| 项目 | 详情 |
|------|------|
| 文件 | `backend/app/services/business_service.py` |
| 函数 | `update_entity` (L237-L268) |

更新时有额外的**磁盘目录重命名**逻辑：

1. **计算旧目录名** (L249): `_compute_dirname(model, entity, session)` → `sanitize_name(name)`
2. **应用变更到内存** (L253-L255): `setattr(entity, key, value)` — 不 commit
3. **计算新目录名** (L258): 基于新 name 值
4. **名称变更 → 重命名磁盘** (L261-L262): `_rename_entity_dir(...)` (L80-L172)
   - `os.replace(old_dir, new_dir)` — 原子操作 (L120)
   - 更新关联 Asset 的 `file_path` 字段 (L158-L166)
   - `auto_commit=False` — 暂存变更，不立即 commit
5. **统一 commit** (L265): 实体改名 + Asset 路径更新在同一事务

#### 步骤 7：删除（DELETE）特殊处理 — delete_entity

| 项目 | 详情 |
|------|------|
| 文件 | `backend/app/services/business_service.py` |
| 函数 | `delete_entity` (L271-L340) |

删除时执行 4 层清理：

1. **收集关联 Asset** (L287-L316): 按 `target_type + target_id` 查询
2. **删除 Asset** (L319-L322): 逐个调用 `delete_asset(session, id, delete_file=True, auto_commit=False)`
   - 清除所有外键引用（Character.image_asset_id 等）(`asset_service.py:155-161`)
   - 物理删除磁盘文件 (`asset_service.py:190-213`)
   - 删除 prompt 文件 (`asset_service.py:205-213`)
3. **rmtree 实体目录** (L325-L333): `shutil.rmtree(entity_dir, ignore_errors=True)` — 兜底清理
4. **删除 DB 记录** (L336-L340): `session.delete(entity)` → `session.commit()`

### 错误处理

| 阶段 | 错误类型 | 处理方式 |
|------|----------|----------|
| ky 请求 | 网络错误 / 超时 | ky 对 GET 自动重试 2 次（`client.ts:29-32`）；POST/PATCH/DELETE 不重试 |
| API 路由 | Pydantic 验证失败 | FastAPI 返回 422 + 详情 |
| API 路由 | 实体不存在 | `HTTPException(404, detail={error: "not_found"})` (`characters.py:55,69,86`) |
| business_service | 目录重命名失败 | 记日志，不回滚 DB（磁盘可后续修复）(`business_service.py:126-127`) |
| useMutation | 任何错误 | `onError` 回调 → `toast.error(e.message)` (`useBusiness.ts:37`) |

### TanStack Query Invalidation 模式

```
创建: qc.invalidateQueries({ queryKey: charKeys.list(projectId) })
      → ["characters", "list", projectId] 前缀匹配
      → 自动 refetch useCharacters() 查询

更新: 同上
删除: 同上
```

## Chain 2：AI 图片生成（角色主图）

### 总览

```
用户点击「生成图片」
  → EntityCard onGenerate 回调
    → GenerateDialog 弹窗
      → 选择 Provider + 配置参数
        → handleSubmit()
          → useGenerate mutation
            → POST /api/generate
              → generation_service.create_task → DB commit
                → _spawn_task(execute_task_async)
                  → 立即返回 { task_id, status: "running" }
                    → WS push task.created
  ─── 以下为异步后台执行 ───
  execute_task_async:
    → update_task_status(running, 5%)
      → 读取上下文（prompt/项目路径/Provider）
        → handler.generate_image(model, prompt, params)
          → async_download_file(url, output_path)
            → create_asset_record → DB commit
              → _auto_fill_entity (image_asset_id)
                → update_task_status(succeeded, 100%)
                  → WS push task.completed { asset_id }
                    → invalidateQueries(["tasks"])
                      → useCharacters refetch
                        → EntityCard 显示新图
```

## Chain 3：剧本解析

### 总览

```
用户点击「解析」
  → ScriptPage 组件
    → useParseScript mutation
      → POST /api/projects/{id}/script/parse
        → api_parse_script (script.py)
          → mark_parsing(doc.id)
            → BackgroundTasks.add_task(parse_script_async)
              → 立即返回 { script_id, status: "parsing" }
  ─── 以下为后台异步执行 ───
  parse_script_async:
    → WS push script.parsing { stage: "reading" }
      → 读取 ScriptDocument.raw_text
        → WS push script.parsing { stage: "parsing" }
          → _llm_parse(raw_text)
            → httpx POST → LLM API (JSON mode)
              → 解析 JSON → 标准化结果
                → WS push script.parsing { stage: "writing" }
                  → write_parsed_to_db (写入 5 张业务表)

                      → WS push script.completed { stats }
                        → invalidateQueries(["script"])
                          → UI 刷新显示解析结果
```

```python
# ws/routes.py:21-53
class ConnectionManager:
    _channels: Dict[str, Set[WebSocket]]  # channel → set of WebSocket

    async def broadcast(self, channel, message):
        text = json.dumps(message, ensure_ascii=False, default=str)
        for ws in self._channels[channel]:
            try:
                await ws.send_text(text)
            except Exception:
                dead.append(ws)
        # 清理断开的连接
        for ws in dead:
            self._channels[channel].discard(ws)
```

- 按频道分组管理连接
- 广播时自动清理失败连接
- `default=str` 处理 datetime 等非 JSON 原生类型
