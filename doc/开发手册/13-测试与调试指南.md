### Python 客户端

```python
import asyncio
import websockets
import json

async def listen():
    async with websockets.connect("ws://127.0.0.1:8000/ws/logs") as ws:
        while True:
            msg = await ws.recv()
            print(json.loads(msg))

asyncio.run(listen())
```

### 单独测试 generate_image（mock httpx）

```python
import asyncio
from unittest.mock import AsyncMock, MagicMock, patch
from app.providers.agnes_handler import AgnesHandler

handler = AgnesHandler(api_key="test-key")

mock_response = MagicMock()
mock_response.status_code = 200
mock_response.json.return_value = {"data": [{"url": "https://cdn.example.com/img.png"}]}

async def test():
    with patch("httpx.AsyncClient") as mock_client_cls:
        mock_client = AsyncMock()
        mock_client.post.return_value = mock_response
        mock_client.__aenter__ = AsyncMock(return_value=mock_client)
        mock_client.__aexit__ = AsyncMock(return_value=False)
        mock_client_cls.return_value = mock_client

        urls = await handler.generate_image(
            model="agnes-image-2.1-flash",
            prompt="一只猫",
            params={"size": "1024x1024"},
        )
        print(f"URLs: {urls}")

asyncio.run(test())
```

## 9. 常见调试场景

| 场景 | 工具 |
| --- | --- |
| API 404 | Swagger 看实际路由；后端启动日志看 "Route registered" |
| API 500 | `GET /api/logs?level=ERROR` 找 traceback |
| WS 收不到消息 | DevTools Network → WS 看收发；LogViewer 看 WsLogHandler 异常 |
| 前端白屏 | 浏览器 Console；LogViewer；可能路由配错 |
| 任务卡住 | 看 `generation_tasks` 表的 status + 任务 logs |
| 文件锁 | 关后端 → 删 WAL → 重启 |
| 图片不显示 | 看 `/assets/<file>` 路径能不能直接访问 |
| 端口占用 | `Get-NetTCPConnection -LocalPort 8000` 找 PID |
| Provider 401/403 | API Key 错；`api_key_encrypted` 解密失败；密钥变了 |
| Provider 404 | 模型名错；`get_capabilities().model` 列出可用模型 |

### 测试结果解读

| pytest 输出 | 含义 | 行动 |
|------------|------|------|
| `X passed` | 全部通过 | 继续下一步 |
| `X failed` | 有失败 | 查看 FAILED 详情，修复代码 |
| `X error` | 测试本身有错误 | 检查 fixture 和 import |
| `X skipped` | 被跳过 | 检查 skip 条件 |

### 调试技巧

```powershell
# 只运行失败的测试
pytest tests/ --lf -v

# 运行到第一个失败就停止
pytest tests/ -x -v

# 显示 print 输出
pytest tests/ -s -v

# 显示每个测试的详细时间
pytest tests/ --durations=10

# 查看覆盖率报告
pytest tests/ --cov=app --cov-report=html
# 然后打开 htmlcov/index.html
```
