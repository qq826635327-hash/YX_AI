## 1. 测试框架与环境

### 1.1 后端测试框架

| 组件 | 选型 | 说明 |
|------|------|------|
| 测试框架 | pytest 7.x | Python 标准测试框架 |
| HTTP 客户端 | httpx.AsyncClient | 异步 HTTP 请求，支持 ASGI |
| 异步支持 | pytest-asyncio | pytest 异步测试支持 |
| Mock 工具 | unittest.mock / pytest-mock | 函数/方法级别 Mock |
| 数据库 | SQLite 内存数据库 | 测试隔离，每次测试全新数据库 |
| 覆盖率 | pytest-cov | 代码覆盖率统计 |
| 工厂 | factory-boy | 测试数据工厂 |
| 断言 | pytest 内置 assert | 简洁的断言语法 |

### 1.2 前端测试框架

| 组件 | 选型 | 说明 |
|------|------|------|
| 测试框架 | Vitest | Vite 原生测试框架 |

### 1.3 测试环境配置

#### 后端 conftest.py

```python
# tests/conftest.py
import pytest
import pytest_asyncio
from httpx import AsyncClient, ASGITransport
from sqlmodel import SQLModel, Session, create_engine
from sqlmodel.pool import StaticPool

from app.main import app
from app.db import get_session

# 内存 SQLite 数据库（测试专用）
TEST_DATABASE_URL = "sqlite+pysqlite:///:memory:"

@pytest.fixture(scope="session")
def engine():
    """创建测试数据库引擎"""
    engine = create_engine(
        TEST_DATABASE_URL,
        connect_args={"check_same_thread": False},
        poolclass=StaticPool,
    )
    SQLModel.metadata.create_all(engine)
    return engine

@pytest.fixture(name="connection")
def connection_fixture(engine):
    """每个测试使用独立连接，支持事务隔离"""
    with engine.connect() as connection:
        yield connection

@pytest.fixture(name="session")
def session_fixture(connection):
    """每个测试在事务中运行，测试后回滚，确保隔离"""
    transaction = connection.begin()
    with Session(bind=connection) as session:
        yield session
    transaction.rollback()

@pytest_asyncio.fixture
async def client(session: Session):
    """异步 HTTP 客户端"""
    from sqlalchemy.orm import close_all_sessions
    async with AsyncClient(
        transport=ASGITransport(app=app),
        base_url="http://test"
    ) as ac:
        # 覆盖数据库依赖（使用事务中的会话）
        app.dependency_overrides[get_session] = lambda: session
        yield ac
        app.dependency_overrides.clear()
    close_all_sessions()
```

### 2.2 单元测试（Unit Tests）

**覆盖范围**:
- Service 层业务逻辑
- Schema 数据验证
- 工具函数（字符串处理、路径计算、加密等）
- 自定义异常处理

**原则**:
- 完全隔离，不依赖外部服务
- 使用 Mock 替代数据库和外部 API
- 每个测试独立，无顺序依赖
- 目标覆盖率 >= 80%

```python
# tests/unit/test_character_service.py
import pytest
from app.services.business_service import (
    list_by_project, get_one, create_entity, update_entity, delete_entity
)
from app.models import Character

class TestCharacterService:
    """角色业务逻辑单元测试"""

    def test_create_character_success(self, session, project):
        """TC-UNIT-CHAR-001: 正常创建角色"""
        data = {"name": "主角A", "char_type": "protagonist"}
        result = create_entity(
            session, Character,
            project_id=project.id, **data
        )
        assert result.name == "主角A"
        assert result.project_id == project.id

    def test_list_characters_by_project(self, session, project, character):
        """TC-UNIT-CHAR-002: 按项目获取角色列表"""
        results = list_by_project(session, Character, project.id)
        assert len(results) >= 1
        assert any(c.name == character.name for c in results)

    def test_get_one_character(self, session, project, character):
        """TC-UNIT-CHAR-003: 获取单个角色"""
        result = get_one(session, Character, character.id, project.id)
        assert result is not None
        assert result.name == character.name

    def test_delete_character(self, session, project, character):
        """TC-UNIT-CHAR-004: 删除角色"""
        delete_entity(session, Character, character.id, project.id)
        result = get_one(session, Character, character.id, project.id)
        assert result is None
```

### 2.3 API 集成测试（Integration Tests）

**原则**:

```python
# tests/api/test_projects.py
import pytest
from httpx import AsyncClient

class TestProjectAPI:
    """项目管理 API 集成测试"""

    @pytest.mark.asyncio
    async def test_create_project(self, client: AsyncClient):
        """TC-API-PROJ-001: 创建项目 - 正常路径"""
        response = await client.post("/api/projects", json={
            "name": "测试项目",
            "description": "自动化测试项目"
        })
        assert response.status_code == 200
        data = response.json()["data"]
        assert data["name"] == "测试项目"
        assert data["status"] == "active"
```

### 9.2 优先级定义

| 优先级 | 定义 | 说明 | 执行策略 |
|--------|------|------|----------|
| **P0** | 阻塞级 | 核心功能，不通过则阻止发布 | 每次提交必跑 |
| **P1** | 严重级 | 重要功能，影响用户体验 | 每日构建必跑 |
| **P2** | 一般级 | 边界条件，异常处理 | 发布前全量运行 |
| **P3** | 轻微级 | 优化项，不影响核心功能 | 有时间时运行 |

### 9.3 优先级分配原则

```
P0 - 必须覆盖:
├── 所有 CRUD 正常路径
├── 所有 404 错误处理
├── 必填参数校验
├── 核心业务流程（创建项目→剧本→生成）
├── WebSocket 连接和基本消息
└── 登录/认证（如有）

P1 - 重要覆盖:
├── 参数边界值（空值、超长、特殊字符）
├── 状态转换验证
├── 级联操作（删除含关联的资源）
├── 分页与排序
├── 前端错误状态展示
└── WebSocket 多客户端

P2 - 扩展覆盖:
├── 并发/竞态条件
├── 性能边界（大量数据）
├── 罕见边界条件
├── 前端 Loading 状态
└── 缓存策略验证

P3 - 可选覆盖:
├── UI 细节（间距、颜色）
├── 极端数据量
├── 国际化字符
└── 无障碍访问
```
