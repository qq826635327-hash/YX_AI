### 3.5 道具管理 API

#### 3.5.1 完整 CRUD 测试

| 编号 | 名称 | 前置条件 | 步骤 | 预期结果 | 优先级 |
|------|------|----------|------|----------|--------|
| TC-API-PROP-001 | 创建道具-正常 | 项目已创建 | POST {"name": "神秘信封"} | 200 | P0 |
| TC-API-PROP-002 | 创建道具-缺少名称 | 项目已创建 | POST {"description": "..."} | 422 | P0 |
| TC-API-PROP-003 | 创建道具-项目不存在 | 无 | POST /api/projects/99999/props | 404 | P0 |
| TC-API-PROP-010 | 获取道具列表 | 项目含道具 | GET /props | 200, 返回道具列表 | P0 |
| TC-API-PROP-011 | 获取道具列表-空 | 项目无道具 | GET /props | 200, [] | P0 |
| TC-API-PROP-020 | 获取道具详情 | 道具已创建 | GET /props/{pid} | 200 | P0 |
| TC-API-PROP-021 | 道具不存在 | 无 | GET /props/99999 | 404 | P0 |
| TC-API-PROP-030 | 更新道具 | 道具已创建 | PATCH {"name": "新名称"} | 200 | P0 |
| TC-API-PROP-031 | 更新不存在道具 | 无 | PATCH /props/99999 | 404 | P0 |
| TC-API-PROP-040 | 删除道具 | 道具已创建 | DELETE /props/{pid} | 200/204 | P0 |
| TC-API-PROP-041 | 删除不存在道具 | 无 | DELETE /props/99999 | 404 | P0 |
| TC-API-PROP-042 | 删除被引用道具 | 道具已关联分镜 | DELETE /props/{pid} | 级联处理或 409 | P1 |

```python
class TestPropAPI:
    """道具管理 API 测试集"""

    @pytest.mark.asyncio
    async def test_tc_api_prop_001_create(
        self, client: AsyncClient, project
    ):
        """TC-API-PROP-001: 创建道具"""
        response = await client.post(
            f"/api/projects/{project.id}/props",
            json={
                "name": "神秘信封",
                "description": "一封来自未知发件人的信件"
            }
        )
        assert response.status_code == 200
        data = response.json()["data"]
        assert data["name"] == "神秘信封"
```
