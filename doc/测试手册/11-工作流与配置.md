# 工作流与配置 API 测试用例

> 本文件包含工作流配置、系统配置和日志 API 的所有测试用例，对应原文 §3.12 + §3.13 + §3.14。

---

### 3.12 工作流配置 API

| 编号 | 名称 | 前置条件 | 步骤 | 预期结果 | 优先级 |
|------|------|----------|------|----------|--------|
| TC-API-WF-001 | 创建工作流 | 无 | POST 有效工作流数据 | 200 | P0 |
| TC-API-WF-002 | 缺少名称 | 无 | POST 缺少 name | 422 | P0 |
| TC-API-WF-003 | 无效 JSON | 无 | POST workflow_json 格式错误 | 422 | P1 |
| TC-API-WF-010 | 获取工作流列表 | 有工作流 | GET /api/config/workflows | 200 | P0 |
| TC-API-WF-020 | 获取工作流详情 | 工作流存在 | GET /api/config/workflows/{id} | 200 | P0 |
| TC-API-WF-021 | 不存在 | 无 | GET /api/config/workflows/99999 | 404 | P0 |
| TC-API-WF-030 | 更新工作流 | 工作流存在 | PATCH 有效数据 | 200 | P0 |
| TC-API-WF-040 | 删除工作流 | 工作流存在 | DELETE /api/config/workflows/{id} | 200/204 | P0 |

### 3.13 系统配置 API

| 编号 | 名称 | 前置条件 | 步骤 | 预期结果 | 优先级 |
|------|------|----------|------|----------|--------|
| TC-API-CFG-001 | 获取系统配置 | 系统已初始化 | GET /api/config/system | 200, 返回配置项 | P0 |
| TC-API-CFG-002 | 获取 ComfyUI 配置 | ComfyUI 已配置 | GET /api/config/comfyui | 200 | P1 |
| TC-API-CFG-003 | 更新 LLM 配置 | 系统已初始化 | PATCH /api/config/llm | 200 | P0 |
| TC-API-CFG-004 | 更新 LLM-无效参数 | 无 | PATCH 无效数据 | 422 | P1 |

---

### 3.14 日志 API

| 编号 | 名称 | 前置条件 | 步骤 | 预期结果 | 优先级 |
|------|------|----------|------|----------|--------|
| TC-API-LOG-001 | 获取日志列表 | 系统有日志 | GET /api/logs | 200, 返回日志 | P0 |
| TC-API-LOG-002 | 获取 INFO 日志 | 有各级日志 | GET /api/logs/info | 200, 仅 INFO 级别 | P1 |
| TC-API-LOG-003 | 清空日志 | 有日志 | POST /api/logs/clear | 200, 日志已清空 | P1 |
| TC-API-LOG-004 | 日志分页 | 大量日志 | GET /api/logs?page=1&size=50 | 200, 返回50条 | P2 |
