# 14 场景

## 1. 路由

`/projects/:id/scenes`

## 2. 与"角色"模块同构

整体 UI 布局、组件、生成对话框、提示词助手、参考图、关联资产等机制都和角色一致；区别在数据字段：

| 字段 | 角色 | 场景 |
| --- | --- | --- |
| 名称 | ✓ | ✓ |
| 简介 | ✓ | ✓ |
| 标签 | ✓ | ✓ |
| 形象/主图 | ✓ (形象.png) | ✓ (概念图.png) |
| 参考图 | ✓ | ✓ |
| 性别 / 年龄 | ✓ | — |
| **cameraHint** | — | ✓（镜头建议：远景 / 全景 / 中景 / 室内 / 室外 / 时间） |
| **关联分镜** | linkedShots | linkedShots |
| 关联道具 | linkedProps | — |

## 3. 概念图（场景的"形象"）

同"形象生成"流程，但提示词偏环境：

- `scenePrompt` = `{{name}} {{cameraHint}} {{appearancePrompt}} {{style}}`

## 4. 镜头建议

场景级提示，AI 在抽分镜时优先考虑：

- `室内 / 室外`
- `时间：日 / 夜 / 黄昏 / 雨`
- `空间：大 / 小 / 走廊 / 庭院`
- `氛围：紧张 / 温馨 / 悬疑`

这些字段会作为"场景上下文"注入到分镜生成 prompt。

## 5. 从剧本导入

解析 剧本.json 的 scenes[].heading，自动建场景卡片。

## 6. 关键 IPC

同 13-角色。
