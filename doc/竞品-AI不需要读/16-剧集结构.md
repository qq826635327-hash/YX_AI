# 16 剧集结构

## 1. 概念

一个项目可以包含多个"剧集"，对应多集短剧。剧集是分镜 / 剧本的父级。

## 2. 剧集清单

`剧集/剧集.json`：

```json
{
  "version": 1,
  "items": [
    { "id": "uuid", "name": "剧集001", "order": 1, "episodeNumber": 1,
      "summary": "…", "scriptPath": "…", "parsedScriptPath": "…",
      "shotsDir": "…", "shotCount": 32, "createdAt": "…", "updatedAt": "…" }
  ]
}
```

## 3. 剧集级目录

```
剧集/剧集001/
├── 剧本.md
├── 剧本.json
├── 分镜/
│   ├── 001.json
│   ├── 001_首帧.png
│   ├── 001_尾帧.png
│   ├── 002.json
│   └── …
└── 日志/                       # 可选：每集独立日志
```

## 4. 操作

- 新建 / 改名 / 排序 / 复制 / 删除
- 集号（episodeNumber）必须唯一
- 集号可从 1 自动递增

## 5. 删除

弹窗提示：
- 关联资产：是否会同步删除角色 / 场景 / 道具？默认"否"
- 关联任务：任务中心会标记"已删除"
- 关联画布：画布里指向该剧集分镜的卡片会标红但保留

## 6. 复刻提示

剧集是一个 **1:N** 的父节点，复刻时建议把"剧集"抽象成一个通用 `Section` 类型，未来支持"按章节"或"按场景"分组。
