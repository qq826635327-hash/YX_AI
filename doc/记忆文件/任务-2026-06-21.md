## 任务三：首尾帧生成视频功能

**目标**：实现分镜首尾帧生成视频功能，接入 Agnes Video V2.0 API，支持图床上传公网 URL。

| `backend/app/db.py` | 迁移：Asset 表新增 3 列 |

| `backend/config.yaml` | 新增 `image_hosting` 配置段 |
| `backend/app/services/image_hosting_service.py` | 新增：SM.MS 图床上传 + Asset 缓存查询/更新（基于 SHA256 hash） |
| `backend/app/providers/agnes_handler.py` | 新增 `agnes-video-v2.0` 模型 + `_translate_video()` / `generate_video()` 异步轮询 |
| `backend/app/providers/base.py` | 新增 `generate_video()` 抽象方法 |
| `backend/app/tasks/execute_task.py` | 视频生成分支 + 参考图按 `reference_images_need_url` 分流（base64 vs 图床） |

### 核心设计

1. **参考图分流**：`ModelCapabilities.reference_images_need_url`
   - `False`（默认）：本地图片 → base64 Data URI → 直接传 API
   - `True`：本地图片 → SM.MS 图床 → 公网 URL → 传 API

2. **图床缓存**：Asset 表存储 `public_url` + `public_url_file_hash`
   - 同一文件不重复上传（SHA256 hash 匹配则复用）
   - 文件变更时自动重新上传

3. **Agnes 视频 API**：异步任务模式
   - POST `/v1/videos` → 返回 `video_id`
   - GET `/agnesapi?video_id=xxx` → 轮询（5s 间隔，10min 超时）
   - `status=completed` 时从 `remixed_from_video_id` 取视频 URL

4. **视频参数**：`agnes-video-v2.0` 支持 width/height/num_frames/frame_rate
   - num_frames 必须满足 8n+1（81/121/161/241/441）
   - 单图走 `image` 字段（图生视频），多图走 `extra_body.image` + `mode=keyframes`（首尾帧关键帧）

- 前端 TypeScript：无错误
- 后端 Import 验证：OK
- DB 迁移：OK

## 任务十一：修复 start.bat 启动失败并启动前后端

**目标**：修复 `D:\影序AI\start.bat` 无法启动前后端的问题，并成功启动后端（8000）和前端（5173）。

**结果**：✅ 完成

### 修改文件

| 文件 | 修改内容 |
|------|----------|
| `start.bat` | 重写启动脚本：修复编码/特殊字符解析问题，用 `ping` 替代 `timeout` 避免 PowerShell 重定向报错，用子程序封装端口检查与等待逻辑，修正路径引号与竖线转义 |

### 核心问题

1. **原 bat 编码问题**：文件为 UTF-8 without BOM，在中文 Windows（默认 GBK 代码页）下被错误解析，导致中文注释/输出被当成命令执行。
2. **竖线字符未转义**：标题框使用 ASCII `\|` 会被 CMD 解析为管道符，触发“命令语法不正确”。
3. **`timeout` 在 PowerShell 中重定向报错**：`timeout /t 1 /nobreak >nul` 经 PowerShell 调用时会提示“不支持输入重新定向”。
4. **路径末尾反斜杠导致引号转义**：`%~dp0` 返回的路径以 `\` 结尾，与后续 `"` 组合会产生 `\"` 转义问题。

### 修复方案

- 将 `start.bat` 保存为 **GBK（936）编码**，并显式 `chcp 936`，确保中文路径与提示正常显示。
- 所有 `echo` 中的竖线改用 `^|` 转义。
- 用 `ping -n N 127.0.0.1 >nul` 替代 `timeout` 实现等待。
- 启动命令中路径不加引号（`%ROOT%backend`），避免 `"` 转义；因路径无空格，无需引号。
- 端口检查逻辑提取为 `:check_and_release_port` 子程序。

### 验证结果

- `start.bat` 在 PowerShell 中直接运行通过，无语法错误。
- 后端健康检查：`http://127.0.0.1:8000/api/health` 返回 `{"status":"ok",...}`。
- 前端页面：`http://127.0.0.1:5173/` 返回 Vite dev server 的 HTML。
- 端口 8000 / 5173 均正常监听。

**状态**：已完成
