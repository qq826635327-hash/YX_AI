## 任务 1：性能监控全链路漏洞修复

### 目标
按真实场景重新审查性能监控实现，修复数据丢失、并发、重复、泄漏等漏洞，确保 AI 自动监控稳定可用。

### 结果
已修复：
- 前端页面关闭/隐藏时 sendBeacon + 同步 XHR 兜底上报，避免数据丢失
- 前端 flush 增加并发锁与 pendingFlush 调度，防止竞态导致事件被拆/丢失
- 前端失败回写限制最大保留事件数，避免无限累积
- 前端 observer 增加 stopObservers / disconnect / clearInterval，防止内存泄漏与重复启动
- longtask / paint 开启 buffered 模式，捕获已发生的性能事件
- 后端 PerfEvent 增加 `(session_id, seq)` 唯一约束，防止重复事件入库
- 后端摄入时先按 seq 内存去重，再写入，唯一冲突时回滚重试
- 后端修复旧事件删除 SQL：只删除已有的旧事件，不会误删新事件
- 后端 p95 计算由 `int` 改为 `ceil`，避免小样本偏差
- 后端 WebSocket 告警推送失败隔离，不影响 API 返回
- 前端 env 读取兼容 SSR / 非 Vite 环境
