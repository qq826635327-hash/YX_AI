# AI Drama Studio · 启动说明

> 版本：v1.0 | 2026-06-21

---

## 一键启动

### Windows

```
双击 start.bat
```

脚本会自动执行：环境检查 → 端口清理 → 启动后端 → 健康验证 → 启动前端 → 打开浏览器。

### macOS / Linux

```bash
bash start.sh
```

按 `Ctrl+C` 停止所有服务。

---

## 一键停止

### Windows

```
双击 stop.bat
```

自动关闭后端和前端进程。

### macOS / Linux

在启动终端中按 `Ctrl+C`。

## 启动流程详解

```
start.bat 执行步骤：

  [1/5] 环境检查
    ├── 检查 Python 3.11+ 是否安装
    ├── 检查 Node.js 18+ 是否安装
    ├── 检查后端虚拟环境 (.venv) 是否存在
    └── 检查前端依赖 (node_modules) 是否安装（缺失则自动 npm install）

  [2/5] 端口检查
    ├── 检查 8000 端口是否被占用（占用则自动释放）
    └── 检查 5173 端口是否被占用（占用则自动释放）

  [3/5] 启动后端
    ├── 以最小化窗口启动 uvicorn
    └── 等待健康检查通过（最多 15 秒）

  [4/5] 启动前端
    ├── 以最小化窗口启动 Vite
    └── 等待 HTTP 响应（最多 10 秒）

  [5/5] 自动打开浏览器
```

### Windows 特殊说明

- 推荐使用 Windows 10/11
- 首次运行前建议先执行 `scripts\init-backend.bat` 初始化虚拟环境
- 如端口被占用，脚本会自动尝试释放；若失败请手动执行 `netstat -ano | findstr :8000` 查找 PID

## 常见问题

### 端口被占用

```powershell
# 查看占用端口的进程
netstat -ano | findstr :8000
netstat -ano | findstr :5173

# 强制终止（替换 <PID>）
taskkill /PID <PID> /F
```

### 后端启动失败

```powershell
# 检查虚拟环境
cd D:\影序AI\backend
.venv\Scripts\python.exe --version

# 重新安装依赖
.venv\Scripts\pip.exe install -e .
```

### 前端启动失败

```powershell
# 重新安装依赖
cd D:\影序AI\frontend
rm -r node_modules
npm install
```

### 数据库锁定

```powershell
# 关闭后端后删除 WAL 文件
del D:\影序AI\data\db\app.sqlite-wal
del D:\影序AI\data\db\app.sqlite-shm
```

## 生产模式

如需部署到生产环境：

```powershell
# 1. 构建前端
cd D:\影序AI\frontend
npm run build

# 2. 启动后端（自动托管 frontend/dist/）
cd D:\影序AI\backend
.venv\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

然后访问 `http://localhost:8000` 即可（前端由后端静态托管）。
